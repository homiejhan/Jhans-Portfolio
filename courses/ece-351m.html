Every time you stream music, make a phone call, take a photo, get an MRI, or use GPS navigation, digital signal processing is running invisibly beneath the surface. The algorithms that compress your audio, cancel echo on your call, sharpen your photo, reconstruct images from X-ray measurements, and separate satellite signals from noise were all designed by engineers who understood how to represent, analyze, and manipulate signals mathematically.

ECE 351M builds the theoretical foundations of DSP from the ground up: starting with the mathematics of discrete-time systems, moving through sampling theory and frequency analysis, and culminating in practical filter design methods and modern topics including adaptive filtering, neural networks, and statistical signal processing. The course bridges the gap between abstract mathematics — Z-transforms, convolution, spectral density — and the algorithms running on every digital device you own.

By the end, the frequency domain is no longer abstract. When you see a filter specification or a spectrum plot, you will understand every design choice behind it — and know how to build the system that produces it.

Why This Matters

Linear Time-Invariant systems are the mathematical foundation of all of digital signal processing. Nearly every transform, filter, and algorithm in this course exists because we are working within the LTI framework — a framework that makes analysis tractable, gives us superposition, and guarantees that the frequency-domain description completely characterizes the system. Understanding what makes a system LTI, and what breaks that property, is prerequisite to everything else. Lectures 1–4 build this foundation from scratch: discrete-time signals, system properties, convolution, the DTFT, and the Z-transform.

Discrete-Time Sequences (Lecture 1)

A discrete-time sequence \(x[n]\) is obtained by sampling a continuous-time signal \(x_a(t)\) at sampling period \(T\): \(x[n] = x_a(nT)\). The sampling frequency is \(f_s = 1/T\). For example, audio signals have bandwidth 0–20 kHz and are typically sampled at \(f_s = 44.1\) ksamples/s (\(T = 22.7\,\mu\text{s}\)).

Four fundamental sequences appear throughout the course:

  • Unit sample (impulse): \(\delta[n] = \begin{cases}1 & n=0 \\ 0 & n \neq 0\end{cases}\). Any arbitrary sequence can be written as \(x[n] = \sum_{k=-\infty}^{\infty} x[k]\,\delta[n-k]\) — a sum of scaled, delayed impulses. This identity is the key to deriving the convolution formula.
  • Unit step: \(u[n] = \begin{cases}1 & n \geq 0 \\ 0 & n < 0\end{cases}\). Note that \(u[n] = \sum_{k=0}^{\infty}\delta[n-k]\) and \(\delta[n] = u[n] - u[n-1]\).
  • Real exponential: \(x[n] = A\alpha^n\). Decays to zero for \(|\alpha| < 1\), grows unbounded for \(|\alpha| > 1\).
  • Complex exponential / sinusoidal: \(x[n] = |A|e^{j(\omega_0 n + \phi)} = |A|\cos(\omega_0 n + \phi) + j|A|\sin(\omega_0 n + \phi)\). Here \(\omega_0\) is the frequency in radians/sample and \(\phi\) is the phase in radians. Because \(e^{j(\omega_0 + 2\pi)n} = e^{j\omega_0 n}\), the frequency is unique only on a \(2\pi\)-interval, typically \(-\pi < \omega_0 \leq \pi\).

A sequence is periodic with period \(N\) if \(x[n+N] = x[n]\) for all \(n\), where \(N\) is a positive integer. For a complex exponential \(e^{j\omega_0 n}\) to be periodic, we need \(\omega_0 N = 2\pi k\) for some integer \(k\). This means \(\omega_0/2\pi\) must be a rational number. Unlike continuous-time, not all discrete-time complex exponentials are periodic — for example, \(x[n] = A\cos\!\left(\frac{\sqrt{2}\pi n}{4}\right)\) is not periodic because \(\omega_0 = \sqrt{2}\pi/4\) leads to \(N = 4\sqrt{2}\,k\), which is never an integer.

Basic sequence operations: sum \(y[n] = x_1[n] + x_2[n]\), product \(y[n] = x_1[n] \cdot x_2[n]\), scalar multiplication \(y[n] = ax[n]\), and delay \(y[n] = x[n-D]\).

System Properties (Lectures 1–2)

A discrete-time system is a transformation \(T\{\cdot\}\) mapping input \(x[n]\) to output \(y[n]\). Five properties must be tested for every system:

  • Memoryless: \(y[n]\) depends only on \(x[n]\) at the current time index — not on past or future inputs. Example: \(y[n] = (x[n])^2\) is memoryless. The delay system \(y[n] = x[n-1]\) has memory.
  • Linear (Superposition): Must satisfy both additivity — \(T\{x_1[n]+x_2[n]\} = T\{x_1[n]\}+T\{x_2[n]\}\) — and homogeneity — \(T\{ax[n]\} = aT\{x[n]\}\). Combined: \(T\{a_1x_1[n]+a_2x_2[n]\} = a_1y_1[n]+a_2y_2[n]\). The accumulator \(y[n]=\sum_{k=-\infty}^n x[k]\) is linear; \(y[n] = \log_{10}(x[n])\) is not (fails at \(x[n]=0\)).
  • Time-Invariant: A delayed input \(x[n-D]\) produces exactly the delayed output \(y[n-D]\), for all \(D\). The accumulator is time-invariant. The decimator \(y[n] = x[Mn]\) is not time-invariant because downsampling changes the time index non-uniformly.
  • Causal: For every \(n\), \(y[n]\) depends only on \(x[n], x[n-1], x[n-2], \ldots\) (present and past). Example: \(y[n] = x[n] - x[n-1]\) is causal; \(y[n] = x[n] - x[n+1]\) is not causal (requires future input).
  • BIBO Stable: If \(|x[n]| \leq B_x < \infty\) for all \(n\), then \(|y[n]| \leq B_y < \infty\) for all \(n\). Example: \(y[n]=(x[n])^2\) is BIBO stable (\(|y[n]| \leq B_x^2\)); \(y[n] = \log_{10}(x[n])\) is not (unbounded at \(x[n] \to 0\)).

LTI Systems & Convolution (Lecture 2)

An LTI system is both linear and time-invariant. Because any input can be written as \(x[n] = \sum_{k=-\infty}^{\infty} x[k]\,\delta[n-k]\), applying linearity and time-invariance in sequence gives: \[ y[n] = T\!\left\{\sum_{k=-\infty}^{\infty} x[k]\,\delta[n-k]\right\} = \sum_{k=-\infty}^{\infty} x[k]\,T\{\delta[n-k]\} = \sum_{k=-\infty}^{\infty} x[k]\,h[n-k] \] where \(h[n] = T\{\delta[n]\}\) is the impulse response. This is the convolution sum: \[ y[n] = x[n] * h[n] = \sum_{k=-\infty}^{\infty} x[k]\,h[n-k] \] Given \(h[n]\), we can compute \(y[n]\) for any input \(x[n]\). Convolution is commutative, associative, and distributive. Cascade connection gives \(h[n] = h_1[n] * h_2[n]\); parallel connection gives \(h[n] = h_1[n] + h_2[n]\).

From the impulse response, the two key LTI properties follow directly:

  • BIBO stability of an LTI system: \(\displaystyle\sum_{k=-\infty}^{\infty}|h[k]| < \infty\). Proof: \(|y[n]| \leq B_x \sum|h[k]|\), so if the sum is finite the output is bounded.
  • Causality of an LTI system: \(h[n] = 0\) for \(n < 0\). This follows because if \(h[-1] \neq 0\) then \(y[n]\) would depend on \(x[n+1]\), a future input.

An LTI system with a finite-length impulse response is an FIR system. One with an infinite-length impulse response is an IIR system. Example: the differentiator \(y[n]=x[n]-x[n-1]\) has \(h[n]=\delta[n]-\delta[n-1]\) — FIR, stable, causal. The accumulator \(y[n]=\sum_{k=-\infty}^n x[k]\) has \(h[n]=u[n]\) — IIR, causal, but not stable (since \(\sum_{k=0}^{\infty}1 = \infty\)). The accumulator and differentiator are inverses of each other because \(u[n]*(\delta[n]-\delta[n-1]) = u[n]-u[n-1] = \delta[n]\).

Representations of LTI Systems (Lectures 2–3)

LTI systems have four equivalent representations: impulse response \(h[n]\), linear difference equation (LDE), frequency response \(H(e^{j\omega})\), and Z-transform \(H(z)\).

Linear Difference Equations (LDE). Any causal LTI system can be written as: \[ \sum_{k=0}^{N} a_k\,y[n-k] = \sum_{m=0}^{M} b_m\,x[n-m] \] Assuming \(a_0 = 1\), this rearranges to a recursive implementation: \(y[n] = \sum_{m=0}^{M} b_m x[n-m] - \sum_{k=1}^{N} a_k y[n-k]\). This is hardware-friendly — only adders, multipliers, and unit delays (D blocks) are required. Example: the accumulator's infinite sum \(\sum_{k=-\infty}^n x[k]\) is impossible to implement directly, but its LDE \(y[n] = y[n-1] + x[n]\) requires only one delay and one adder.

Frequency Response & Eigenfunctions. The complex exponential \(e^{j\omega n}\) is an eigenfunction of every LTI system. When \(x[n] = e^{j\omega n}\): \[ y[n] = \sum_{k=-\infty}^{\infty} h[k]\,e^{j\omega(n-k)} = e^{j\omega n}\underbrace{\sum_{k=-\infty}^{\infty} h[k]\,e^{-j\omega k}}_{H(e^{j\omega})} = H(e^{j\omega})\,e^{j\omega n} \] The output is the same frequency, scaled by the complex number \(H(e^{j\omega})\) — the frequency response. This means an LTI system cannot create new frequencies.

DTFT. For any sequence \(x[n]\), the Discrete-Time Fourier Transform is: \[ X(e^{j\omega}) = \sum_{n=-\infty}^{\infty} x[n]\,e^{-j\omega n}, \quad -\pi \leq \omega \leq \pi \] The DTFT is periodic with period \(2\pi\) and exists if \(\sum_{n=-\infty}^{\infty}|x[n]| < \infty\). The inverse DTFT recovers the sequence: \[ x[n] = \frac{1}{2\pi}\int_{-\pi}^{\pi} X(e^{j\omega})\,e^{j\omega n}\,d\omega \] Key DTFT pairs: \(a^n u[n] \xleftrightarrow{\mathcal{F}} \frac{1}{1-ae^{-j\omega}}\) for \(|a|<1\); \(\delta[n] \xleftrightarrow{\mathcal{F}} 1\); \(\delta[n-n_0] \xleftrightarrow{\mathcal{F}} e^{-j\omega n_0}\). Key theorems: linearity, time-shift \(x[n-n_d] \xleftrightarrow{\mathcal{F}} e^{-j\omega n_d}X(e^{j\omega})\), convolution \(x[n]*y[n] \xleftrightarrow{\mathcal{F}} X(e^{j\omega})Y(e^{j\omega})\), and Parseval's theorem \(\sum_{n=-\infty}^{\infty}|x[n]|^2 = \frac{1}{2\pi}\int_{-\pi}^{\pi}|X(e^{j\omega})|^2\,d\omega\).

The IDTFT of an ideal lowpass filter \(H(e^{j\omega})=1\) for \(|\omega|\leq\omega_c\), 0 otherwise, is \(h[n] = \frac{\sin(\omega_c n)}{\pi n}\) — a two-sided infinite sinc function. This is the starting point for FIR window-method design (Topic 6).

The Z-Transform (Lecture 4)

The Z-transform generalizes the DTFT by replacing \(e^{j\omega}\) with the complex variable \(z\): \[ X(z) = \sum_{n=-\infty}^{\infty} x[n]\,z^{-n} \] When the DTFT exists, \(X(e^{j\omega}) = X(z)\big|_{z=e^{j\omega}}\) — the Z-transform evaluated on the unit circle. The Z-transform requires a Region of Convergence (ROC): the set of \(z\) where \(|X(z)| < \infty\). The ROC has no poles inside it.

ROC shapes depend on the signal's sidedness:

  • Right-sided (causal): ROC is \(|z| > r\) (outside a circle). Example: \(a^n u[n] \xrightarrow{\mathcal{Z}} \frac{1}{1-az^{-1}}\), ROC: \(|z|>|a|\).
  • Left-sided (anti-causal): ROC is \(|z| < r\) (inside a circle). Example: \(-a^n u[-n-1] \xrightarrow{\mathcal{Z}} \frac{1}{1-az^{-1}}\), ROC: \(|z|<|a|\).
  • Two-sided: ROC is a ring (annulus) — the intersection of the right-sided and left-sided ROCs. If the ROCs do not intersect, the Z-transform does not exist.

For an LTI system described by the LDE above, the Z-transform gives the system function: \[ H(z) = \frac{Y(z)}{X(z)} = \frac{\sum_{m=0}^{M} b_m z^{-m}}{\sum_{k=0}^{N} a_k z^{-k}} = \frac{b_0(1-c_1 z^{-1})(1-c_2 z^{-1})\cdots(1-c_M z^{-1})}{a_0(1-d_1 z^{-1})(1-d_2 z^{-1})\cdots(1-d_N z^{-1})} \] The \(c_i\) are zeros (numerator roots) and the \(d_i\) are poles (denominator roots). The combination of pole-zero locations and the ROC uniquely specifies the system.

Three critical stability/causality connections from the Z-domain:

  • For a stable LTI system, the unit circle must lie in the ROC (since \(H(e^{j\omega})\) must be bounded).
  • For a causal system, the ROC extends outside the outermost pole: \(|z| > \max_k|d_k|\).
  • For a system that is both causal and stable, all poles must lie strictly inside the unit circle \(|z| < 1\).

Inverse Z-transforms are typically computed via partial fraction expansion. For example, \(X(z) = \frac{1}{(1-az^{-1})(1-bz^{-1})}\) with \(|a|,|b|<1\) expands to \(\frac{a/(a-b)}{1-az^{-1}} - \frac{b/(a-b)}{1-bz^{-1}}\), giving \(x[n] = \frac{a}{a-b}a^n u[n] - \frac{b}{a-b}b^n u[n]\) for the causal ROC.

Block Diagram Representations of LTI Systems (Lecture 5)

Every LDE can be drawn as a signal-flow network using three primitive building blocks: an adder (sums two signals), a scalar multiplier (scales by constant \(a\)), and a unit delay (\(z^{-1}\) block, shifts by one sample: \(x[n] \to x[n-1]\)). In the Z-domain a chain of \(k\) delays equals multiplication by \(z^{-k}\).

Direct Form I implements the LDE directly with a feedforward section \(H_f(z)\) for the input side and a feedback section \(H_b(z)\) for the output side. For the general LDE \(\sum_{k=0}^{N}a_k y[n-k] = \sum_{k=0}^{M}b_k x[n-k]\), the transfer function is:

\[ H(z) = \frac{\sum_{k=0}^{M} b_k z^{-k}}{\sum_{k=0}^{N} a_k z^{-k}} \]

Direct Form I requires \(M + N\) delay elements — \(M\) in the feedforward path and \(N\) in the feedback path. It is the most memory-intensive structure.

Direct Form II (Canonic Form) exploits commutativity of LTI systems to swap \(H_f\) and \(H_b\), then merge their identical delay chains into one shared chain. The intermediate signal \(w[n]\) satisfies:

\[ w[n] = x[n] + a_1 w[n-1] + \cdots + a_N w[n-N] \] \[ y[n] = b_0 w[n] + b_1 w[n-1] + \cdots + b_M w[n-M] \]

Direct Form II uses only \(\max(N, M)\) delays — the theoretical minimum. The tradeoff is loss of access to the intermediate signal and tighter dynamic range constraints (internal signals can overflow even when input/output are bounded).

Cascade (Biquad) Form factors \(H(z)\) as a product of second-order sections:

\[ H(z) = \prod_{k=1}^{N_s} \frac{b_{0k} + b_{1k}z^{-1} + b_{2k}z^{-2}}{1 - a_{1k}z^{-1} - a_{2k}z^{-2}} \]

Each section \(H_k(z)\) is implemented in Direct Form II with 2 delays. First-order sections are a special case with \(a_2 = b_2 = 0\). Cascade form is modular, numerically robust (poles and zeros are paired locally), and is the standard hardware structure for IIR filters.

Parallel Form uses partial fraction expansion to write \(H(z)\) as a sum of second-order sections:

\[ H(z) = \sum_{k=1}^{N_p} \frac{b_{0k} + b_{1k}z^{-1}}{1 - a_{1k}z^{-1} - a_{2k}z^{-2}} \]

All sections run on the same input simultaneously and their outputs are summed. Useful for parallel hardware implementations.

Minimum Phase Systems. The inverse system \(H_i(z) = 1/H(z)\) is causal and stable only if all zeros of \(H(z)\) lie inside the unit circle (since they become poles of \(H_i\)). Combined with the requirement that all poles lie inside the unit circle for \(H(z)\) itself, a system that is causal and stable AND has a causal, stable inverse is called a minimum phase system:

\[ H_i(z) = \frac{1}{H(z)}, \quad \text{both } H(z) \text{ and } H_i(z) \text{ causal \& stable} \iff \text{all poles and zeros inside unit circle} \]

From Block Diagram to \(H(z)\). Two techniques extract the transfer function from a block diagram:

  • Auxiliary variable method: label an internal node \(W(z)\), write equations at each summing junction, then solve for \(Y(z)/X(z)\). For a feedback loop with forward path \(F(z)\) and feedback path \(G(z)\): \[ H(z) = \frac{F(z)}{1 - F(z)G(z)} \]
  • Structural modification: push a block through a branch point (duplicating it) to convert a complex topology into recognizable cascade, parallel, or feedback sub-structures that can be reduced using standard rules.

Key Takeaways

  • A discrete-time complex exponential \(e^{j\omega_0 n}\) is periodic only if \(\omega_0/(2\pi)\) is rational — test by checking if \(\omega_0 N = 2\pi k\) has integer solutions.
  • To prove a system is NOT linear/time-invariant/stable, find a single counterexample. To prove it IS, you must show the property holds for all inputs.
  • The accumulator \(y[n] = \sum_{k=-\infty}^n x[k]\) is causal and linear but NOT stable (\(\sum|h[k]| = \infty\)). The differentiator \(y[n]=x[n]-x[n-1]\) is causal, linear, and stable. They are inverses.
  • The convolution formula \(y[n] = \sum_k x[k]h[n-k]\) is derived from the impulse decomposition identity \(x[n] = \sum_k x[k]\delta[n-k]\) combined with linearity and time-invariance — know this derivation.
  • The DTFT exists if \(\sum|x[n]| < \infty\); otherwise the Z-transform may still exist for some ROC. If the ROC includes the unit circle, the DTFT equals the Z-transform on the unit circle.
  • For a causal, stable system: all poles of \(H(z)\) must be inside the unit circle. This is the single most important stability test in filter design.
  • Always specify the ROC with the Z-transform — the same rational function \(X(z)\) can correspond to different time-domain signals (causal vs. anti-causal) depending on the ROC.
  • Cascaded LTI systems: \(H(z) = H_1(z) \cdot H_2(z)\); parallel LTI systems: \(H(z) = H_1(z) + H_2(z)\).
  • Direct Form I uses \(M + N\) delays; Direct Form II uses only \(\max(N, M)\) — the theoretical minimum — by sharing one delay chain between the feedback and feedforward paths.
  • Cascade (biquad) form is the standard IIR implementation: numerically stable, modular, pairs poles with nearby zeros to minimize coefficient sensitivity.
  • A minimum phase system has all poles AND zeros strictly inside the unit circle — it is causal, stable, and its inverse \(H_i(z) = 1/H(z)\) is also causal and stable.
  • Feedback loop formula: \(H(z) = F(z)/(1 - F(z)G(z))\) — memorize the derivation; it appears in digital control loops, echo cancellers, and PLLs.
  • To find \(H(z)\) from a block diagram: either assign an auxiliary variable \(W(z)\) at an internal node and solve the junction equations, or use structural modification to expose standard cascade/parallel/feedback patterns.

Real-World Applications

Every digital filter in audio production, telecommunications, medical imaging, and control systems is an LTI system. The convolution sum is exactly what a DSP chip computes in a tight multiply-accumulate loop. The Z-transform pole-zero plot is how engineers in industry (Prof. Telang's background includes Broadcom, where this is daily practice) diagnose filter stability and shape frequency response without computing the full DTFT. Audio CD sampling at 44.1 kHz and optical transceivers sampling at 200 Gsamp/s both reduce to the same discrete-time mathematics. Linear difference equations allow infinite-memory systems (like the accumulator) to be implemented with finite hardware — a single delay register and adder replace an infinite sum.

Why This Matters

Every digital signal processing system begins with sampling — the bridge between the continuous-time physical world and the discrete-time digital domain. Sampling theory answers the fundamental question: when can you perfectly reconstruct a continuous-time signal from a discrete set of samples, and what happens when you cannot? The consequences of getting this wrong — aliasing — are irreversible and ubiquitous in practice. This is the theoretical foundation of every ADC, every audio interface, every digital camera, and every software-defined radio.

The Two-Step Sampling Model (Lecture 6)

Sampling a continuous-time signal \(x_c(t)\) at period \(T\) is analyzed as a two-step process.

Step 1 — Modulation by an impulse train. The periodic impulse train \(s(t) = \sum_{n=-\infty}^{\infty} \delta(t - nT)\) has a Fourier Transform that is itself an impulse train in frequency: \(S(j\Omega) = \frac{2\pi}{T}\sum_{k=-\infty}^{\infty}\delta(\Omega - k\Omega_s)\), where \(\Omega_s = 2\pi/T\). Multiplying \(x_c(t)\) by \(s(t)\) gives:

\[ x_s(t) = x_c(t) \cdot s(t) = \sum_{n=-\infty}^{\infty} x_c(nT)\,\delta(t - nT) \]

Note that \(x_s(t)\) is still a continuous-time signal. In the frequency domain, multiplication in time equals convolution in frequency:

\[ X_s(j\Omega) = \frac{1}{T} \sum_{k=-\infty}^{\infty} X_c\!\left(j(\Omega - k\Omega_s)\right) \]

This is the central result: the spectrum of the sampled signal is the original spectrum \(X_c\) repeated at every integer multiple of \(\Omega_s\), scaled by \(1/T\).

Step 2 — Conversion to a discrete-time sequence. Defining \(x[n] = x_c(nT)\) and comparing with the DTFT \(X(e^{j\omega}) = \sum_n x[n]e^{-j\omega n}\) shows that \(X_s(j\Omega) = X(e^{j\omega})\big|_{\omega=\Omega T}\). Substituting \(\omega = \Omega T\) into the Step 1 result:

\[ X(e^{j\omega}) = \frac{1}{T} \sum_{k=-\infty}^{\infty} X_c\!\left(j\!\left(\frac{\omega}{T} - \frac{2\pi k}{T}\right)\right) \]

The digital frequency \(\omega = \Omega T\) is dimensionless (rad/sample). The sampling frequency always maps to \(\omega_s = 2\pi\), and \(\omega = \pi\) always corresponds to \(\Omega_s/2 = \pi/T\) (half the sampling rate), regardless of \(T\).

Images are the repeated copies of \(X_c\) appearing in \(X(e^{j\omega})\). Sinusoids at frequencies \(f_0,\, f_0+f_s,\, f_0+2f_s,\,\ldots\) all produce identical sample sequences — they are indistinguishable after sampling. For example, with \(f_s = 150\,\text{Hz}\), sinusoids at 50 Hz, 200 Hz, 350 Hz, and 500 Hz all map to the same discrete-time samples.

Aliasing (Lecture 6)

For spectral images to be separable they must not overlap. The non-overlap condition is:

\[ \Omega_N \leq \Omega_s - \Omega_N \quad\Longrightarrow\quad \Omega_s \geq 2\Omega_N \]

When this is violated (\(\Omega_s < 2\Omega_N\)), adjacent copies overlap — aliasing. A tone at \(\Omega_0\) with \(\Omega_s/2 < \Omega_0 < \Omega_s\) aliases to \(\Omega_s - \Omega_0\):

\[ \cos(\Omega_0 t) \;\xrightarrow{\text{sample at } \Omega_s,\;\Omega_0 > \Omega_s/2}\; \cos\!\left((\Omega_s - \Omega_0)\,t\right) \]

Aliasing is irreversible — the alias overlaps baseband content and cannot be separated. In the time domain: a 100 Hz cosine sampled at \(f_s = 150\,\text{Hz}\) (\(f_s/2 = 75\,\text{Hz}\)) produces samples identical to a 50 Hz cosine, since \(f_s - f_0 = 150 - 100 = 50\,\text{Hz}\).

Prevention requires an anti-aliasing filter — a continuous-time LPF cutting off all energy above \(\Omega_s/2\) — applied to \(x_c(t)\) before the ADC. No post-processing can recover aliased content.

Nyquist-Shannon Sampling Theorem (Lecture 7)

Let \(x_c(t)\) be bandlimited with \(X_c(j\Omega) = 0\) for \(|\Omega| > \Omega_N\). Then \(x_c(t)\) is uniquely determined by its samples \(x[n] = x_c(nT)\) if and only if:

\[ \Omega_s = \frac{2\pi}{T} \geq 2\Omega_N \]

The minimum permissible sampling rate \(2\Omega_N\) is the Nyquist rate. The maximum representable signal bandwidth \(\Omega_N = \Omega_s/2\) is the Nyquist bandwidth. In frequency units: \(f_s \geq 2f_N\). For audio (up to 20 kHz), the CD standard uses \(f_s = 44.1\,\text{kHz}\), sampled well above Nyquist with margin for a realizable anti-aliasing filter rolloff.

Signal Reconstruction (Lecture 7)

D/C conversion (reconstruction) is also analyzed in two steps.

Step 1: Convert the sequence back to an impulse train: \(x_s(t) = \sum_{n=-\infty}^{\infty} x[n]\,\delta(t - nT)\).

Step 2 — Ideal reconstruction filter. Apply an ideal lowpass filter \(H_r(j\Omega)\) with gain \(T\) and cutoff \(\pi/T\):

\[ H_r(j\Omega) = \begin{cases} T & |\Omega| \leq \pi/T \\ 0 & \text{otherwise} \end{cases}, \qquad h_r(t) = \frac{\sin(\pi t/T)}{\pi t/T} \]

This passes only the baseband copy (\(k=0\)) of \(X_s\), discarding all images. The reconstruction in the time domain is:

\[ x_r(t) = \sum_{n=-\infty}^{\infty} x[n]\, h_r(t - nT) = \sum_{n=-\infty}^{\infty} x[n]\, \frac{\sin(\pi(t-nT)/T)}{\pi(t-nT)/T} \]

This is sinc interpolation: each sample \(x[n]\) is weighted by a sinc pulse centered at \(t = nT\). When \(\Omega_s > 2\Omega_N\) the reconstruction is exact: \(x_r(t) = x_c(t)\).

In practice, ideal sinc interpolation is non-causal and infinite-length. Real DACs use a zero-order hold (step-wise output) followed by an analog smoothing filter. Linear interpolation uses a triangle pulse for \(h_r(t)\) — causal and simple, but introduces frequency-dependent distortion since the triangle's spectrum is \(\text{sinc}^2\), not rectangular.

Discrete-Time Processing of Analog Signals (Lecture 8)

The standard architecture samples \(x_c(t)\) at rate \(f_s\), applies a digital filter \(H(e^{j\omega})\), then reconstructs. If \(x_c(t)\) is bandlimited and sampled above Nyquist, the combined system behaves as a single continuous-time filter with effective frequency response:

\[ H_{\text{eff}}(j\Omega) = \begin{cases} H(e^{j\Omega T}) & |\Omega| < \pi/T \\ 0 & |\Omega| \geq \pi/T \end{cases} \]

To design a digital filter that achieves a continuous-time cutoff \(f_c\), convert: \(\omega_c = 2\pi f_c / f_s\). For example, a signal bandlimited to 2000 Hz sampled at \(f_s = 5000\,\text{Hz}\), with a digital LPF cutting at \(f_c = 1750\,\text{Hz}\), requires digital cutoff \(\omega_c = 2\pi \times 1750/5000 = 0.7\pi\,\text{rad/sample}\).

Downsampling (Lecture 8)

Downsampling by \(M\) (sampling rate compression) keeps every \(M\)th sample:

\[ x_d[n] = x[Mn] = x_c[nMT] \]

The new sampling period is \(T_d = MT\). The DTFT of the downsampled signal is:

\[ X_d(e^{j\omega}) = \frac{1}{M} \sum_{i=0}^{M-1} X\!\left(e^{j(\omega - 2\pi i)/M}\right) \]

Two steps visually: (1) superpose \(M\) shifted copies of \(X(e^{j\omega})\) separated by \(2\pi/M\); (2) expand the frequency axis by \(M\). To avoid aliasing in the downsampled signal, the original bandwidth must satisfy \(\omega_N \leq \pi/M\). If it exceeds this, apply a discrete-time LPF with cutoff \(\pi/M\) first:

\[ x[n] \;\longrightarrow\; \text{LPF}\!\left(\omega_c = \tfrac{\pi}{M}\right) \;\longrightarrow\; \downarrow M \;\longrightarrow\; x_d[n] \]

Aliasing after downsampling is fundamentally the same phenomenon as aliasing in original sampling — both arise from insufficient sampling rate relative to signal bandwidth.

Upsampling (Lecture 8)

Upsampling by \(L\) (sampling rate expansion) inserts \(L-1\) zeros between each sample:

\[ x_u[n] = \begin{cases} x[n/L] & n = 0, \pm L, \pm 2L, \ldots \\ 0 & \text{otherwise} \end{cases} \]

The DTFT compresses the frequency axis by \(L\): \(X_u(e^{j\omega}) = X(e^{j\omega L})\). This squeezes the baseband to \([-\pi/L, \pi/L]\) and creates spectral images at multiples of \(2\pi/L\). A discrete-time LPF with cutoff \(\pi/L\) and gain \(L\) removes the images:

\[ x[n] \;\longrightarrow\; \uparrow L \;\longrightarrow\; \text{LPF}\!\left(\omega_c = \tfrac{\pi}{L},\;\text{gain}=L\right) \;\longrightarrow\; x_u[n] \]

Rational rate conversion. To change the sampling rate by \(L/M\), upsample by \(L\) first, apply a single LPF with cutoff \(\min(\pi/L,\,\pi/M)\), then downsample by \(M\). The upsampling step must come first to avoid losing spectral content. This allows any rational rate change entirely in discrete time — no analog conversion required. For example, converting 44.1 kHz (CD) to 48 kHz (DAT) uses \(L/M = 160/147\).

Key Takeaways

  • Sampling creates infinite spectral copies (images) of \(X_c(j\Omega)\) spaced \(\Omega_s\) apart — the Nyquist condition \(\Omega_s \geq 2\Omega_N\) ensures non-overlap.
  • Aliasing is irreversible — the anti-aliasing LPF must come before the ADC; no post-processing can recover overlapped content.
  • Aliased frequency for a tone at \(\Omega_0 > \Omega_s/2\): the alias appears at \(\Omega_s - \Omega_0\). Example: 100 Hz at \(f_s = 150\,\text{Hz}\) aliases to 50 Hz.
  • \(\omega = \Omega T\) (rad/sample); \(\omega = \pi\) always equals \(f_s/2\). Convert digital cutoff to physical: \(f_c = \omega_c f_s / (2\pi)\).
  • Ideal reconstruction: ideal LPF with gain \(T\) and cutoff \(\pi/T\); impulse response is \(\text{sinc}(t/T)\); exact when Nyquist condition holds.
  • Downsampling by \(M\): keep every \(M\)th sample; spectrum superimposes \(M\) shifted copies then expands by \(M\); must LPF to \(\pi/M\) first to prevent aliasing.
  • Upsampling by \(L\): insert \(L-1\) zeros; spectrum compresses by \(L\) and images appear; must LPF at \(\pi/L\) after to remove images.
  • Rational rate conversion \(L/M\): upsample first (\(\uparrow L\)), apply LPF at \(\min(\pi/L,\pi/M)\), then downsample (\(\downarrow M\)) — all in discrete time.

Real-World Applications

Every audio interface (24-bit, 192 kHz), every smartphone camera sensor, every software-defined radio, and every medical imaging system is a sampling system. CD audio (44.1 kHz) captures hearing up to 20 kHz with margin for a realizable anti-aliasing filter. Sigma-delta ADCs oversample by 64–512×, enabling 24-bit resolution from simple 1-bit comparators. Multirate DSP enables speech codecs to operate at 8 kHz while the microphone samples at 48 kHz. Sample rate conversion (e.g., 44.1 kHz → 48 kHz in professional audio) uses rational upsampling/downsampling (\(L/M = 160/147\)) with a polyphase filter bank. Aliasing is intentionally exploited in subsampled radar and undersampled RF receivers to frequency-shift signals without analog mixers.

Topic 3 Discrete Fourier Transform (DFT)
›

Why This Matters

The DTFT gives us a theoretically beautiful but computationally impossible frequency representation — it requires an infinite sum and produces a continuous function of frequency. The DFT solves both problems: it takes a finite-length sequence and produces a finite set of frequency coefficients, enabling digital computation. It is the practical tool that makes spectrum analysis, filtering, and spectral estimation possible on real hardware. Every audio equalizer, speech recognizer, and wireless modem is built on DFT mathematics.

Concepts

The N-point DFT of a length-\(N\) sequence \(x[n]\) is: \[ X[k] = \sum_{n=0}^{N-1} x[n]\, e^{-j2\pi kn/N}, \quad k = 0, 1, \ldots, N-1 \] and the inverse DFT (IDFT) recovers the original sequence: \[ x[n] = \frac{1}{N} \sum_{k=0}^{N-1} X[k]\, e^{j2\pi kn/N}, \quad n = 0, 1, \ldots, N-1 \] The DFT bin \(k\) corresponds to frequency \(\omega_k = 2\pi k/N\) (rad/sample), or \(f_k = k f_s/N\) (Hz). Bins \(k = 0, 1, \ldots, N/2\) represent \(0\) to \(f_s/2\); bins \(k = N/2+1, \ldots, N-1\) represent negative frequencies \(-f_s/2\) to \(-f_s/N\).

Using the notation \(W_N = e^{-j2\pi/N}\) (the twiddle factor), the DFT becomes: \[ X[k] = \sum_{n=0}^{N-1} x[n]\, W_N^{kn} \] Direct computation costs \(O(N^2)\) complex multiplications.

Key DFT properties (all operations modulo \(N\)):

  • Linearity: \(\text{DFT}\{ax[n]+bw[n]\} = aX[k]+bW[k]\)
  • Circular shift: \(\text{DFT}\{x[(n-m)_N]\} = W_N^{km}\,X[k]\)
  • Circular convolution: \(\text{DFT}\{x[n] \circledast y[n]\} = X[k]\,Y[k]\)
  • Parseval's theorem: \(\sum_{n=0}^{N-1}|x[n]|^2 = \frac{1}{N}\sum_{k=0}^{N-1}|X[k]|^2\)
  • Symmetry (real input): \(X[N-k] = X^*[k]\) — spectrum is conjugate-symmetric, so only \(N/2+1\) bins are unique

Circular convolution arises because the DFT implicitly treats signals as periodic with period \(N\). To compute linear convolution of two length-\(N\) sequences using the DFT, zero-pad both to length \(\geq 2N-1\) before taking the DFT — this is the basis of fast convolution. The overlap-add and overlap-save methods extend this to filtering long (or infinite) sequences in blocks.

Spectral leakage occurs when a signal is not exactly periodic in the analysis window — energy from a single frequency spreads across many bins. Leakage is reduced by windowing: multiplying \(x[n]\) by a window \(w[n]\) (e.g., Hanning, Hamming, Blackman) before taking the DFT. Windowing reduces leakage at the cost of widening the main lobe (frequency resolution is \(\Delta f = f_s/N\); increasing \(N\) improves resolution). The zero-padding trick: appending zeros before the DFT interpolates the spectrum but does not improve true frequency resolution — it only reveals what resolution is already present.

Key Takeaways

  • DFT bin \(k\) corresponds to frequency \(f_k = k f_s / N\) Hz — the \(N/2+1\) bins from \(0\) to \(f_s/2\) are the unique bins for real input
  • Circular convolution ≠ linear convolution — always zero-pad to length \(\geq N_1 + N_2 - 1\) before multiplying DFTs
  • Spectral leakage is caused by implicit periodicity assumption — window the signal before the DFT to trade main-lobe width for reduced side lobes
  • Zero-padding interpolates the spectrum (smoother plot) but does NOT improve frequency resolution — resolution is \(\Delta f = f_s/N\)
  • Parseval\'s theorem: total energy is preserved between time and frequency domains — use it to check DFT computations

Real-World Applications

The DFT is used in every spectrum analyzer, audio equalizer, speech recognition frontend, OFDM wireless modem, and medical MRI machine. Overlap-add filtering is how DAWs (digital audio workstations) apply convolution reverb effects in real time on blocks of audio samples. The circular convolution property is exploited in OFDM (Wi-Fi, LTE, 5G) where the cyclic prefix converts linear channel convolution into circular — enabling single-tap frequency-domain equalization. Windowing functions are a routine choice in spectrum analyzers — the Blackman window is preferred for measuring harmonic distortion in audio equipment.

Topic 4 Fast Fourier Transform (FFT)
›

Why This Matters

The DFT requires \(O(N^2)\) operations — at \(N = 1024\) that is over a million multiplications per frame. The FFT reduces this to \(O(N \log_2 N)\) — a 100× speedup for \(N = 1024\), a 1000× speedup for \(N = 65536\). This is not a different transform — it is an efficient algorithm for computing exactly the same DFT result. The FFT made real-time spectral analysis possible on 1960s hardware. Today it enables GPS receivers to correlate pseudorandom codes in milliseconds, MRI scanners to reconstruct 3D images in seconds, and 5G base stations to process hundreds of OFDM subcarriers simultaneously.

Concepts

The Cooley-Tukey FFT exploits the periodicity and symmetry of the twiddle factors \(W_N^{kn}\) to decompose an \(N\)-point DFT into smaller DFTs. The key identity: for \(N = 2M\), split \(x[n]\) into even-indexed and odd-indexed subsequences: \[ X[k] = \underbrace{\sum_{n=0}^{M-1} x[2n]\, W_{2M}^{2kn}}_{X_{\text{even}}[k]} + W_{2M}^k \underbrace{\sum_{n=0}^{M-1} x[2n+1]\, W_{2M}^{2kn}}_{X_{\text{odd}}[k]} \] Since \(W_{2M}^{2k} = W_M^k\), both sums are \(M\)-point DFTs. Applying this recursively for \(N = 2^\nu\) gives \(\nu = \log_2 N\) stages, each requiring \(N/2\) complex multiplications — total cost \(\frac{N}{2}\log_2 N\) multiplications and \(N\log_2 N\) additions.

The butterfly operation is the core computational unit: \[ A' = A + W_N^k B, \qquad B' = A - W_N^k B \] An \(N = 8\) point decimation-in-time (DIT) FFT has \(\log_2 8 = 3\) stages, each with 4 butterflies, totaling 12 complex multiplications versus 64 for direct DFT computation. The bit-reversal permutation reorders the input before the first butterfly stage — the index \(n\) in bit-reversed binary order maps the natural input to the DIT structure.

Two main decomposition strategies: Decimation-in-Time (DIT) — split the input sequence into even/odd halves; Decimation-in-Frequency (DIF) — split the output sequence. Both give the same \(O(N\log N)\) complexity. For non-power-of-2 \(N\), the Bluestein algorithm (chirp-Z transform) or mixed-radix FFTs (radix-3, radix-4) extend FFT efficiency to arbitrary lengths.

The inverse FFT (IFFT) computes the IDFT with the same \(O(N\log N)\) cost — simply conjugate the twiddle factors and scale by \(1/N\). Fast convolution via FFT: zero-pad both \(x[n]\) and \(h[n]\) to length \(N \geq N_x + N_h - 1\), compute FFT of each, multiply pointwise, IFFT — total cost \(O(N\log N)\) versus \(O(N^2)\) for direct convolution. Break-even over direct convolution occurs around \(N \approx 32\).

Key Takeaways

  • FFT cost: \(\frac{N}{2}\log_2 N\) multiplications; direct DFT: \(N^2\) — at \(N = 1024\) the FFT is \(\approx 200\times\) faster
  • FFT computes exactly the same result as the DFT — it is an algorithm, not a different transform
  • Bit-reversal permutation is required for in-place DIT FFT — the input must be reordered before the butterfly stages
  • Fast convolution: FFT both signals (zero-padded), multiply, IFFT — faster than direct convolution for \(N \gtrsim 32\)
  • For non-power-of-2 lengths, always zero-pad to the next power of 2 — the savings from avoiding a mixed-radix FFT almost always outweigh the extra computation

Real-World Applications

The FFT is one of the ten most important algorithms of the 20th century (per IEEE). FFTW (Fastest Fourier Transform in the West) achieves near-optimal FFT performance on any hardware and is embedded in MATLAB, NumPy, and virtually every DSP library. OFDM — the modulation scheme in Wi-Fi, LTE, and 5G — uses IFFT at the transmitter to generate subcarriers and FFT at the receiver to demodulate them. Audio codec MP3 uses the modified DCT (a variant of the FFT) for time-frequency analysis. Computed tomography (CT scanners) uses the FFT to reconstruct images from X-ray projections via the filtered backprojection algorithm.

Topic 5 IIR Filter Design
›

Why This Matters

IIR filters use feedback — the output depends on past output values as well as past and present inputs. This recursive structure enables very sharp frequency selectivity with a small number of coefficients compared to FIR filters. A 6th-order Butterworth IIR filter can achieve the same stopband attenuation as a 60th-order FIR filter. The tradeoff is phase nonlinearity and potential instability. IIR design leverages a century of analog filter theory — Butterworth, Chebyshev, and elliptic filters — and transforms it to the discrete-time domain.

Concepts

An IIR filter is described by an LCCDDE with feedback terms: \[ y[n] = \sum_{k=1}^{N} a_k\, y[n-k] + \sum_{k=0}^{M} b_k\, x[n-k] \] with system function \(H(z) = \frac{\sum_{k=0}^{M} b_k z^{-k}}{1 - \sum_{k=1}^{N} a_k z^{-k}}\). Because the denominator polynomial can be non-trivial, IIR filters have poles, and stability requires all poles inside the unit circle.

The primary design strategy is analog prototype design followed by transformation to discrete time. Step 1: design an analog prototype filter \(H_c(s)\) meeting the specifications. Step 2: apply a frequency transformation to convert to discrete time.

The three classical analog prototype filters:

  • Butterworth: Maximally flat magnitude in the passband. The \(N\)th-order Butterworth magnitude response is \(|H_c(j\Omega)|^2 = \frac{1}{1 + (\Omega/\Omega_c)^{2N}}\). Poles lie on a circle of radius \(\Omega_c\) in the left half \(s\)-plane, equally spaced by \(\pi/N\) radians.
  • Chebyshev Type I: Equiripple in the passband, monotone in the stopband. Achieves steeper rolloff than Butterworth for the same order. The ripple magnitude \(\varepsilon\) is a free parameter.
  • Chebyshev Type II: Monotone in the passband, equiripple in the stopband. Useful when flat passband response is required.
  • Elliptic (Cauer): Equiripple in both passband and stopband. Achieves the sharpest transition for a given filter order — optimal in the minimax sense. At the cost of highly nonlinear phase.

Two methods for converting analog prototypes to discrete-time IIR filters:

Impulse Invariance: set \(h[n] = T\, h_c(nT)\) — the discrete-time impulse response is a sampled version of the analog one. The relationship \(s_k \mapsto z_k = e^{s_k T}\) maps \(s\)-plane poles to \(z\)-plane poles. The frequency mapping is \(j\Omega \mapsto e^{j\Omega T}\), which wraps the entire \(j\Omega\) axis onto the unit circle — causing aliasing of frequency response replicas. Only suitable for lowpass and bandpass filters where the analog response is negligible above \(\Omega_s/2\).

Bilinear Transform: substitute \(s = \frac{2}{T}\cdot\frac{z-1}{z+1}\) (or equivalently \(z = \frac{1 + (T/2)s}{1 - (T/2)s}\)). This maps the entire \(j\Omega\) axis to the unit circle without aliasing via a nonlinear frequency warping: \[ \Omega = \frac{2}{T}\tan\!\left(\frac{\omega}{2}\right) \implies \omega = 2\arctan\!\left(\frac{\Omega T}{2}\right) \] High analog frequencies compress into the band near \(\omega = \pi\). To compensate, prewarping the critical frequency before designing the analog prototype ensures it maps correctly to digital: \(\Omega_c = \frac{2}{T}\tan(\omega_c/2)\). The bilinear transform is the preferred method for lowpass, highpass, bandpass, and notch filters.

Key Takeaways

  • IIR filter stability: all poles of \(H(z)\) must lie strictly inside the unit circle — check after every transformation
  • Bilinear transform is always preferred over impulse invariance — it avoids aliasing entirely; only the prewarping step must not be skipped
  • Prewarp the critical frequency: \(\Omega_c = \frac{2}{T}\tan(\omega_c/2)\) before designing the analog prototype
  • Butterworth: maximally flat, lowest selectivity; Chebyshev: more selective but ripple; Elliptic: highest selectivity but nonlinear phase
  • Implement high-order IIR as cascaded second-order sections (biquads) to minimize coefficient sensitivity and prevent numerical overflow
  • IIR filters are not linear-phase in general — never use them when phase distortion matters (e.g., data communications)

Real-World Applications

IIR filters are used wherever computational efficiency matters more than linear phase: audio equalizers (parametric EQ uses biquad IIR sections), hearing aid processors (real-time filtering on battery power), power system 60 Hz notch filters, and biomedical instrumentation (ECG baseline wander removal). The Butterworth filter design equations are used in analog active filter design (op-amp Sallen-Key topology). The bilinear transform is implemented in virtually every DSP textbook filter design function — MATLAB's butter(), cheby1(), and ellip() all use it internally. Biquad filter sections in fixed-point arithmetic are the workhorses of real-time embedded audio processing.

Topic 6 FIR Filter Design
›

Why This Matters

FIR filters have no feedback — the output depends only on a finite window of past and present inputs. This means they are always stable, and they can be designed to have exactly linear phase — a property IIR filters cannot achieve. Linear phase means all frequency components experience the same time delay, which is critical for any application where waveform shape must be preserved: data communications, audio processing, and biomedical signal acquisition. The design challenge is achieving sharp frequency selectivity with a tractable filter length.

Concepts

An FIR filter of length \(M\) (order \(M-1\)) has the system function: \[ H(z) = \sum_{n=0}^{M-1} h[n]\, z^{-n} \] It has \(M-1\) zeros and no poles (except at \(z = 0\)). All FIR filters are BIBO stable. An FIR filter has linear phase (constant group delay \(\tau = (M-1)/2\)) if and only if \(h[n]\) is symmetric: \(h[n] = h[M-1-n]\) (Type I/II, even/odd \(M\)) or anti-symmetric: \(h[n] = -h[M-1-n]\) (Type III/IV). Linear phase in the frequency domain means \(H(e^{j\omega}) = |H(e^{j\omega})|\, e^{-j\omega(M-1)/2}\).

Window Method: Start with the ideal (infinite) impulse response of the desired filter (e.g., ideal lowpass: \(h_d[n] = \frac{\omega_c}{\pi}\text{sinc}(\omega_c n / \pi)\)) and truncate it to length \(M\) by multiplying by a window \(w[n]\): \[ h[n] = h_d[n]\, w[n], \quad 0 \leq n \leq M-1 \] Common windows and their characteristics:

  • Rectangular: sharpest transition band (\(\approx 0.9/M \cdot f_s\)), but highest sidelobe level (−13 dB) — causes Gibbs phenomenon
  • Hanning: −44 dB first sidelobe, wider transition band (\(\approx 3.1/M \cdot f_s\))
  • Hamming: −53 dB first sidelobe, wider transition (\(\approx 3.3/M \cdot f_s\))
  • Blackman: −74 dB first sidelobe, widest transition (\(\approx 5.5/M \cdot f_s\))
  • Kaiser: parameterized by \(\beta\) — adjustable tradeoff between sidelobe level and transition width; preferred for its flexibility: \(M \approx \frac{A - 8}{2.285\, \Delta\omega}\) where \(A\) is the desired stopband attenuation in dB

Parks-McClellan (Equiripple) Method: Uses the Chebyshev equiripple criterion — the optimal FIR filter minimizes the maximum error between the desired and actual frequency response (minimax criterion). The Remez exchange algorithm iteratively finds the optimal coefficient set. The result has equiripple behavior in both passband and stopband — the minimum-length filter for given specifications. Compared to the window method, Parks-McClellan gives sharper transitions for the same \(M\) and allows independent specification of passband ripple \(\delta_1\) and stopband attenuation \(\delta_2\).

Filter Length Estimation. For the window method (Kaiser): \[ M \approx \frac{A - 8}{2.285\,\Delta\omega} + 1 \] where \(\Delta\omega = \omega_{\text{stop}} - \omega_{\text{pass}}\) (rad/sample) and \(A = -20\log_{10}\delta_s\) dB. For Parks-McClellan (Harris rule of thumb): \[ M \approx \frac{-10\log_{10}(\delta_p \delta_s) - 13}{14.6\,\Delta f/f_s} + 1 \] FIR filters typically require 5–10× more coefficients than an equivalent IIR filter — but their linear phase and stability are often worth the computational cost.

Key Takeaways

  • Linear phase requires symmetric coefficients: \(h[n] = h[M-1-n]\); guaranteed group delay of \((M-1)/2\) samples
  • All FIR filters are stable (no poles off unit circle) — no stability check needed, unlike IIR
  • Window method: easy to design, good intuition, not optimal; Parks-McClellan: optimal, equiripple, minimum length for given specs
  • Gibbs phenomenon limits rectangular window stopband to −21 dB regardless of filter length — use Kaiser or Blackman for high attenuation
  • FIR computational cost scales with length \(M\) — the Kaiser length estimate (\(M \approx (A-8)/(2.285\Delta\omega)\)) is your first design step

Real-World Applications

FIR filters with linear phase are used in every digital communications receiver — matched filters, channel equalizers, and pulse shaping filters all exploit linear phase to avoid inter-symbol interference. Audio crossover networks in studio monitors use linear-phase FIR designs. Medical ECG and EEG systems use FIR notch filters to remove 60 Hz power line interference without distorting the waveform phase. Polyphase FIR structures enable efficient decimation and interpolation in software-defined radios. The Parks-McClellan algorithm is implemented as MATLAB's firpm() and fir1() functions used by DSP engineers daily.

Topic 7 Adaptive Filters & Neural Networks

Why This Matters

Classical filter design assumes we know the signal and noise characteristics in advance. In many real systems we do not — the channel changes, the noise is non-stationary, or the signal of interest is unknown a priori. Adaptive filters solve this by continuously updating their own coefficients based on observed data, converging to the optimal filter for the current environment. From noise-canceling headphones to 5G channel equalizers to hearing aids, adaptive filtering is the technology that makes high-performance signal processing possible in unknown and changing environments.

Concepts

The Wiener filter is the optimal linear filter in the mean-square error sense for stationary signals. Given a desired response \(d[n]\) and input \(x[n]\), the optimal FIR Wiener filter of length \(M\) minimizes: \[ \xi = E\{|e[n]|^2\} = E\{|d[n] - \mathbf{h}^T \mathbf{x}[n]|^2\} \] Setting the gradient to zero gives the Wiener-Hopf equations: \[ \mathbf{R}_{xx}\, \mathbf{h}_{\text{opt}} = \mathbf{r}_{dx} \] where \(\mathbf{R}_{xx}\) is the \(M \times M\) autocorrelation matrix of the input and \(\mathbf{r}_{dx}\) is the cross-correlation vector between the desired signal and input. The optimal filter is \(\mathbf{h}_{\text{opt}} = \mathbf{R}_{xx}^{-1}\, \mathbf{r}_{dx}\). The minimum mean-square error is \(\xi_{\min} = \sigma_d^2 - \mathbf{r}_{dx}^T \mathbf{R}_{xx}^{-1} \mathbf{r}_{dx}\).

The Wiener solution requires knowing \(\mathbf{R}_{xx}\) and \(\mathbf{r}_{dx}\) — not always available, and expensive to invert for large \(M\). Adaptive filters estimate these statistics on-line and update coefficients iteratively.

The LMS (Least Mean Squares) algorithm is a stochastic gradient descent approximation. Using the instantaneous error \(e[n] = d[n] - \mathbf{h}^T[n]\mathbf{x}[n]\), the update rule is: \[ \mathbf{h}[n+1] = \mathbf{h}[n] + \mu\, e[n]\, \mathbf{x}[n] \] The step size \(\mu\) controls the convergence speed–misadjustment tradeoff. LMS converges in the mean if: \[ 0 < \mu < \frac{2}{\lambda_{\max}} \approx \frac{2}{M \sigma_x^2} \] where \(\lambda_{\max}\) is the largest eigenvalue of \(\mathbf{R}_{xx}\). A small \(\mu\) gives accurate but slow convergence; large \(\mu\) converges faster but with more misadjustment (residual error after convergence).

The RLS (Recursive Least Squares) algorithm minimizes the sum of all past squared errors (with exponential forgetting factor \(\lambda \in (0,1]\)): \[ \mathbf{h}[n] = \arg\min_{\mathbf{h}} \sum_{i=0}^{n} \lambda^{n-i} |d[i] - \mathbf{h}^T \mathbf{x}[i]|^2 \] RLS converges in approximately \(2M\) samples (vs. \(M/\mu\sigma_x^2\) for LMS) at the cost of \(O(M^2)\) complexity per sample (vs. \(O(M)\) for LMS). The matrix inversion lemma enables recursive update of \(\mathbf{P}[n] = (\mathbf{R}_{xx}^{(n)})^{-1}\).

Key adaptive filter applications: system identification (model an unknown plant); noise cancellation (subtract a reference noise signal adaptively correlated with the noise on the primary channel); channel equalization (invert an unknown communication channel); linear prediction (predict \(x[n]\) from its past values — used in speech coding LPC).

Neural networks generalize adaptive filtering beyond the linear model. A single neuron computes \(y = f(\mathbf{w}^T \mathbf{x} + b)\) where \(f\) is a nonlinear activation function (sigmoid, ReLU). Multi-layer perceptrons (MLPs) stack these to approximate arbitrary functions. The backpropagation algorithm computes gradients of the loss with respect to all weights via the chain rule, enabling stochastic gradient descent training analogous to LMS. Convolutional neural networks (CNNs) apply learned filter banks — a direct generalization of linear FIR filtering to the nonlinear domain.

Key Takeaways

  • Wiener filter requires knowing \(\mathbf{R}_{xx}\) and \(\mathbf{r}_{dx}\) — if unknown, use LMS/RLS to estimate them adaptively
  • LMS stability condition: \(0 < \mu < 2/\lambda_{\max} \approx 2/(M\sigma_x^2)\) — normalize by input power to get NLMS
  • LMS: \(O(M)\) per sample, slow convergence; RLS: \(O(M^2)\) per sample, fast convergence (~\(2M\) samples) — choose based on application
  • Misadjustment is proportional to \(\mu M \sigma_x^2\) — excess MSE above \(\xi_{\min}\) due to using noisy gradient estimates
  • Neural networks are nonlinear adaptive systems — backpropagation is stochastic gradient descent, the same idea as LMS but applied to multi-layer nonlinear models

Real-World Applications

Noise-canceling headphones (Bose QuietComfort, AirPods Pro) use adaptive LMS filters running at 48 kHz on a DSP chip — the reference microphone picks up ambient noise, and the adaptive filter learns to subtract it from the audio feed. 5G base station receivers use adaptive channel equalizers running RLS variants to track rapidly changing multipath channels. Speech codecs (G.711, AMR) use LPC (linear prediction) — a Wiener filter — to model the vocal tract and achieve high compression. Cochlear implants use adaptive noise suppression to separate speech from background noise. Deep learning audio models (WaveNet, Whisper) are large-scale nonlinear adaptive systems trained with backpropagation on millions of audio samples.

Topic 8 Random Processes & Spectral Estimation

Why This Matters

Real signals are not deterministic — they are random. Noise, speech, seismic data, financial time series, neural recordings — all are best described statistically. Random process theory provides the mathematical framework for characterizing signals whose exact values are unknown but whose statistical properties are stable. Spectral estimation connects this to the frequency domain, answering the question: given a finite observation of a random process, what can we say about its power distribution across frequencies? This is the foundation of radar, sonar, speech processing, neuroscience, and quantitative finance.

Concepts

A random process (or stochastic process) \(\{X[n]\}\) is an indexed family of random variables. For each realization (outcome), we observe a deterministic sequence; the statistical properties are characterized across all realizations. Key statistical descriptors:

  • Mean: \(\mu_X[n] = E\{X[n]\}\)
  • Autocorrelation: \(R_{XX}[n_1, n_2] = E\{X[n_1]X^*[n_2]\}\)
  • Autocovariance: \(C_{XX}[n_1,n_2] = R_{XX}[n_1,n_2] - \mu_X[n_1]\mu_X^*[n_2]\)

A process is wide-sense stationary (WSS) if (1) \(\mu_X[n] = \mu_X\) (constant mean) and (2) \(R_{XX}[n_1,n_2] = R_{XX}[n_1 - n_2]\) depends only on the lag \(k = n_1 - n_2\). WSS is the key assumption enabling spectral analysis — under WSS, the autocorrelation sequence \(R_{XX}[k]\) is a well-defined function of lag alone, and the power spectral density (PSD) is its DTFT: \[ S_{XX}(e^{j\omega}) = \sum_{k=-\infty}^{\infty} R_{XX}[k]\, e^{-j\omega k} \] The PSD is real, non-negative, and \(2\pi\)-periodic. Its integral gives the total power: \(\sigma_X^2 = E\{|X[n]|^2\} = \frac{1}{2\pi}\int_{-\pi}^{\pi} S_{XX}(e^{j\omega})\, d\omega\).

When a WSS process \(X[n]\) is passed through an LTI system with frequency response \(H(e^{j\omega})\), the output \(Y[n]\) is also WSS with: \[ S_{YY}(e^{j\omega}) = |H(e^{j\omega})|^2\, S_{XX}(e^{j\omega}) \] This is the fundamental result connecting LTI systems and random processes. Output power = \(\frac{1}{2\pi}\int |H|^2 S_{XX}\, d\omega\).

Two processes \(X[n]\) and \(Y[n]\) are jointly WSS if the cross-correlation \(R_{XY}[k] = E\{X[n+k]Y^*[n]\}\) depends only on lag \(k\). They are uncorrelated if \(R_{XY}[k] = \mu_X \mu_Y^*\) for all \(k\). A white noise process has PSD \(S_{XX}(e^{j\omega}) = \sigma^2\) (flat — equal power at all frequencies) and autocorrelation \(R_{XX}[k] = \sigma^2 \delta[k]\).

Spectral estimation is the problem of estimating the PSD from a finite observed record \(\{x[0], x[1], \ldots, x[N-1]\}\). Non-parametric methods:

  • Periodogram: \(\hat{S}(e^{j\omega}) = \frac{1}{N}|X_N(e^{j\omega})|^2\) where \(X_N\) is the \(N\)-point DFT. The periodogram is an inconsistent estimator — its variance does not decrease as \(N \to \infty\).
  • Bartlett method: average \(K\) non-overlapping periodograms, each from a segment of length \(L = N/K\). Reduces variance by \(K\) at the cost of frequency resolution \((\Delta f = f_s/L)\).
  • Welch method: extends Bartlett with overlapping (50%) windowed segments. Reduces bias from spectral leakage and further reduces variance. The standard method in practice (implemented as pwelch() in MATLAB).

Parametric methods assume the process is generated by an autoregressive (AR), moving average (MA), or ARMA model and estimate model parameters from data — yielding higher-resolution spectral estimates from short records. The Yule-Walker equations relate AR model parameters to the autocorrelation sequence, enabling maximum entropy spectral estimation.

Key Takeaways

  • WSS requires constant mean and autocorrelation depending only on lag \(k\) — both conditions must hold to apply PSD analysis
  • PSD through LTI: \(S_{YY} = |H|^2 S_{XX}\) — this is the fundamental result; it is why filters shape the spectrum of noise
  • The periodogram is biased and inconsistent — always use Welch\'s method (averaged, windowed periodograms) for reliable spectral estimates
  • White noise has flat PSD \(\sigma^2\) and \(R_{XX}[k] = \sigma^2\delta[k]\) — uncorrelated samples at every lag; the baseline noise model in DSP
  • Resolution–variance tradeoff in spectral estimation is fundamental: more averaging reduces variance but coarsens frequency resolution

Real-World Applications

PSD analysis is used in every field that processes random signals. Radar and sonar use the PSD to detect targets buried in clutter noise. Speech recognition systems analyze the power spectrum (via MFCC features built on Welch-estimated spectra) to identify phonemes. Seismologists use spectral analysis to characterize earthquakes and detect nuclear tests. EEG/ECoG neuroscience research uses Welch PSD estimation to study brain rhythms (alpha, beta, gamma bands). Algorithmic trading systems use autocorrelation and spectral methods to detect periodicities in financial time series. The Kalman filter — covered in this course — is the time-domain counterpart of Wiener filtering for non-stationary processes.
