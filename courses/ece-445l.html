Every device with a microcontroller — your thermostat, your car's engine management system, a medical infusion pump, a drone flight controller — was built by engineers who understood how to bridge the gap between hardware and software. Group 1 Intro to Embedded Systems & ARM Architecture
›

Why This Matters

Every device around you with a microcontroller — your car's engine management unit, a medical infusion pump, a drone flight controller, an industrial robot — was built by engineers who understood both the hardware and the software. ECE 445L exists because that dual fluency is exactly what industry pays a premium for. As McDermott puts it: software-centric students need to become proficient with hardware, and hardware-centric students need to become proficient with software. This first group gives you the foundation everything else is built on.

Concepts

An embedded system is a computer built to perform a dedicated function within a larger device. Unlike a general-purpose PC, it operates under tight constraints — limited memory, fixed power budget, and real-time deadlines. The course hardware is the Texas Instruments TM4C123 LaunchPad, a development board built around the ARM Cortex-M4 — the most widely deployed 32-bit processor architecture on Earth.

The Cortex-M4 is a RISC (Reduced Instruction Set Computer) with a Harvard architecture: separate buses for instructions and data allow simultaneous fetch and memory access. Key features: few opcodes, fixed-length instructions, many general-purpose registers (R0–R15), and a 3-stage pipeline. R13 is the stack pointer, R14 the link register (return address), and R15 the program counter.

The TM4C123 uses memory-mapped I/O — every peripheral register (GPIO, timer, ADC, UART) is assigned a fixed address in the 4 GB address space. Reading and writing peripherals is identical to reading and writing RAM, just at different addresses. The memory map: Flash at 0x0000.0000 (256 KiB), SRAM at 0x2000.0000 (32 KiB).

Fixed-point arithmetic is essential when floating-point is too slow or unavailable. A fixed-point number is represented as an integer I and a fixed constant Δ (the resolution), where the true value = I × Δ. Key design choices: signed vs. unsigned, number of bits (8/16/32), and the resolution Δ. Range is the number of distinguishable values (determined by bit width); resolution is the smallest detectable step (equal to Δ). A practical example: a 12-bit ADC with Vref = 3.3 V gives Δ = 3.3/4096 ≈ 0.000806 V/bit. Storing millivolts as an integer (Δ = 0.001 V): I = (3300 × N) >> 12. Watch for overflow (promoting to 32-bit before multiply) and dropout (underflow from shifting before multiplying).

ESD (Electrostatic Discharge) can instantly destroy a LaunchPad — the human body can accumulate 3,000–5,000 V. Always handle boards by the edges and ground yourself first.

Key Takeaways

  • The Cortex-M4 is RISC/Harvard: few opcodes, fixed-length, separate instruction and data buses, 3-stage pipeline
  • All peripheral control is through memory-mapped registers — the datasheet is the source of truth for every address and bit field
  • Fixed-point value = I × Δ: choose Δ based on the resolution you need, choose bit width based on the range you need
  • Overflow and dropout are the two failure modes of fixed-point arithmetic — always cast to uint32_t before multiplying large values
  • Accuracy (closeness to true value), precision (reproducibility), and resolution (smallest detectable change) are three distinct concepts

Real-World Applications

Fixed-point arithmetic is how virtually every real-time DSP system on embedded hardware works — from the audio codec in your phone to the motor controller in an electric vehicle. The ARM Cortex-M4 is the processor inside products from hearing aids (Cochlear) to automotive ECUs (Bosch) to satellites. Understanding the architecture at the register level is what allows you to debug hardware-software interactions that no simulator can reproduce.

Group 2 Interrupts, Debugging & Critical Sections
›

Why This Matters

Real embedded systems have to respond to multiple asynchronous events simultaneously — a button press, a timer firing, a byte arriving over serial. Without interrupts, the processor wastes every cycle polling. Without debugging skills, you can't observe what the hardware is actually doing versus what you think it's doing. These two topics are the difference between embedded systems that work and embedded systems that seem to work.

Concepts

A thread is the path of action of software as it executes. When an interrupt fires, a new background thread (the ISR) is created. The main program and all ISRs share global variables but each has its own local variables and registers on the stack. The ARM Cortex-M manages interrupt priority through the NVIC (Nested Vectored Interrupt Controller) — with 77 interrupt sources on the TM4C123 across 8 priority levels. A higher-priority ISR can preempt a lower-priority one; same-priority ISRs cannot preempt each other.

Three things are required to use any interrupt: first, arm it — set the peripheral's arm bit to allow that specific flag to trigger an interrupt. Second, enable the interrupt in the NVIC and set its priority. Third, enable globally with I=0 (CPSIE I). The global PRIMASK bit (I) in the program status register enables (I=0) or disables (I=1) all interrupts at once — setting I=1 does not dismiss pending interrupts, it postpones them. Hardware sets the flag when requesting an interrupt; the ISR must clear it (acknowledge) or disarm before returning.

Edge-triggered interrupts fire on a GPIO voltage transition — rising, falling, or both edges. The TM4C123 has 77 interrupt vectors; for example, GPIO Port F is vector 46, IRQ 30, priority bits 23–21 of NVIC_PRI7_R. When Port F fires: the current instruction finishes, 8 registers (R0–R3, R12, LR, PC, PSR) are pushed onto the stack, the vector address is loaded into PC, IPSR is set to 46, and LR is set to 0xFFFFFFF9 (return to Thread mode using MSP).

Switch debouncing is essential because mechanical contacts bounce 1–20 ms on every press, generating dozens of spurious edges. Software debounce: on the first edge, disarm the GPIO interrupt and start a one-shot timer (~10 ms). When the timer fires, re-read the switch state and re-arm the GPIO. Hardware debounce: an RC filter (100 kΩ × 0.1 µF ≈ 10 ms time constant) slows the voltage transition so the processor only sees one clean edge.

Latency is the time between when an event occurs and when the ISR responds. For input devices it's the time between data ready and software reading it; for periodic events (ADC, DAC, control loops) it's the difference between the scheduled time and the actual run time — excess latency generates jitter. With a single interrupt source, jitter comes only from multicycle instructions (±1–2 cycles). With multiple interrupts competing, jitter spreads across tens to thousands of cycles — this is why timer-triggered ADC sampling eliminates jitter that software-triggered sampling cannot.

A critical section exists when: (1) a global variable is shared between two threads, (2) the access is non-atomic (more than one instruction), and (3) at least one thread writes. The classic Lab 3 example: the SysTick ISR updates Hour and Minute, while main reads them to display. If an interrupt fires between reading Hour and reading Minute, the display shows an inconsistent time. Two solutions: make atomic — disable interrupts while reading shared data; or remove sharing — pack Hour and Minute into a single 32-bit variable (Time = 100×Hour + Minute) so the read is one atomic load instruction. A function is reentrant if it can be called simultaneously by two threads without corruption — achieved by using only local variables, or by making global accesses atomic.

Debugging has two forms: performance debugging (timing — is the system meeting its deadlines?) and functional debugging (data — is the computation correct?). The process is: stabilize the bug, localize it, then correct it. Hardware tools: logic analyzers capture multiple digital channels simultaneously and decode serial protocols — nonintrusive, meaning they don't affect system timing. Oscilloscopes show analog voltage vs. time — essential for measuring signal levels and rise times. Software tools: dumps save data into a global array during execution (use instead of printf — far less intrusive); a heartbeat LED or GPIO toggle confirms the program is running; assertions verify invariants in code. Intrusiveness matters: print statements and breakpoints halt the processor and can mask timing bugs — use logic analyzers and dumps instead for real-time systems.

The Central Limit Theorem justifies ADC averaging: if noise is independent and zero-mean, averaging N samples makes the distribution Gaussian and drives the standard deviation toward zero. Averaging 4 samples roughly halves noise; averaging 16 samples reduces it by 4×.

Key Takeaways

  • Three steps for any interrupt: arm the device, enable in NVIC with priority set, enable globally (I=0) — miss any one and the interrupt silently fails
  • The ISR must acknowledge (clear the flag) or disarm before returning — failing to do so causes the ISR to fire repeatedly in an infinite loop
  • Switch bounce is real — always debounce, or one button press generates dozens of events
  • A critical section requires shared global + non-atomic access + at least one write — all three conditions must be present
  • In this class, anytime you want to debug with printf, use a dump and observe the data later — printf is too intrusive for real-time systems
  • Higher-priority ISR (lower number) preempts lower-priority — priority 3 ISR can be interrupted by priority 0, 1, or 2, but not by 3–7

Real-World Applications

Every keyboard, mouse, touchscreen, and button in every consumer device uses edge-triggered interrupts — polling would waste the entire CPU budget waiting for input that arrives unpredictably. Pacemakers use SysTick-style periodic interrupts to deliver precisely timed cardiac pulses — a jitter of even a few milliseconds is clinically significant. The Heisenberg principle of debugging is real: breakpoints in a motor control loop cause the motor to stop, making the bug disappear. Dumps are how firmware engineers debug production systems where attaching a debugger is impossible. The Central Limit Theorem is why your phone's accelerometer averages hundreds of samples before reporting orientation — raw ADC output is too noisy to use directly.

Group 3 Data Acquisition Systems
›

Why This Matters

The physical world is analog. Sensors produce voltages; motors need voltages. The ADC and DAC are the hardware bridges between the digital processor and reality — and their characteristics directly determine what your system can measure and control. Equally important: if you sample incorrectly, you corrupt your data in a way that is mathematically irreversible. Every instrument from a handheld multimeter to a medical ECG to an industrial oscilloscope is fundamentally a data acquisition system.

Concepts

The Nyquist-Shannon Sampling Theorem states: a band-limited signal with no frequency components above W Hz may be completely recovered from samples taken at a rate greater than 2W samples per second. In practice, Valvano's Postulate adds a stricter engineering rule: sample at more than 10× the highest frequency of interest for the reconstructed waveform to visually resemble the original. An anti-aliasing analog lowpass filter must be placed before the ADC to remove all signal content above ½fₛ — content above this frequency folds back into the baseband as aliasing, which is irreversible. A 100 Hz signal sampled at 1600 Hz reconstructs perfectly; the same signal sampled at just above its Nyquist rate looks nothing like the original in the time domain even though it mathematically satisfies the theorem.

A complete Data Acquisition System (DAS) consists of: measurand → transducer → analog preamplifier → analog filter → sample-and-hold → ADC → microcomputer → actuator. Each stage introduces its own errors. Key quantitative DAS parameters: range (rₓ — total span of measurable values), resolution (Δx — smallest detectable change), precision (number of distinguishable alternatives nₓ), repeatability (σ of repeated measurements under same conditions), and reproducibility (σ under different conditions). Qualitative parameters for detection systems: true positive (event occurs, system detects it), false positive (event doesn't occur, system reports it), false negative (event occurs, system misses it), true negative (event doesn't occur, system correctly says so). Sensitivity = TP/(TP+FN); Specificity = TN/(TN+FP).

Three ADC architectures: A Successive Approximation (SAR) ADC performs a binary search — each bit is decided by comparing VIN to a DAC output, retaining or rejecting it. Example: converting 45 (6-bit SAR): Is X≥32? Yes (retain) → 0x1. Is X≥48? No (reject) → 0x10. Is X≥40? Yes → 0x101. Is X≥44? Yes → 0x1011. Is X≥46? No → 0x10110. Is X≥45? Yes → 0x101101 = 45. The TM4C123 uses a 12-bit SAR at up to 1 MSPS — good accuracy, moderate speed. A Flash ADC uses 2ᴺ−1 simultaneous comparators — one clock cycle conversion, extremely fast but area-exponential. A Sigma-Delta ADC oversamples at very high rates with 1-bit quantization then applies a digital decimation filter — excellent resolution for audio (high resolution, low speed), poor for fast distance measurement.

Time jitter in sampling violates the Nyquist theorem (which assumes uniform intervals), introduces errors in derivative computation (dx/dt = (x(t)−x(t−Δt))/Δt), distorts DAC-generated waveforms, and causes instability in control loops. With one interrupt source, jitter spans only ±2–3 clock cycles. With multiple competing interrupts, jitter grows to ±thousands of cycles. Elimination: use timer-triggered ADC sampling (the timer hardware directly triggers the ADC conversion, bypassing software latency entirely).

From Lec 13 on transistor interfaces: BJT operation — in saturation, the transistor acts as a closed switch (Vbe > Vbe(SAT), typically 0.6–1.2 V). Collector current Ic = hfe × Ib. The TIP120 Darlington transistor cascades two BJTs for hfe ≥ 1000, allowing a 3 mA GPIO signal to control 3 A of motor current. MOSFET operation: voltage-controlled current source; on when Vgs > threshold; Rds ≈ 0 when on, Rds = ∞ when off. Digital logic interfacing requires VOH > VIH, IOH > IIH, VOL < VIL, IOL > IIL — the TM4C123 GPIO can drive 2/4/8 mA (configurable). To interface to the ESP8266 (Vih = 2.475 V, Voh(TM4C) = 2.4 V), a level shifter or pull-up resistor is required since Voh barely meets Vih. Inductive loads (motors, solenoids) require a flyback diode across the load to clamp the back-EMF voltage spike when switching off — without it, the spike can permanently destroy the driver transistor.

Key Takeaways

  • Aliasing is irreversible — the anti-aliasing filter must come before the ADC, not after
  • Nyquist says sample >2W; Valvano's practical rule says sample >10× fmax for the waveform to look right on a plot
  • SAR ADC is good for accuracy; Flash ADC is fastest; Sigma-Delta is best for high-resolution audio
  • Timer-triggered ADC completely eliminates sampling jitter — software-triggered sampling cannot
  • Accuracy ≠ precision ≠ resolution — a precise ADC can be inaccurate; a high-resolution ADC can be imprecise
  • Always use a flyback diode across any inductive load — the back-EMF spike will destroy the driver without it
  • TM4C123 to ESP8266 interface requires a level shifter — Voh(TM4C) = 2.4 V barely meets Vih(ESP) = 2.475 V

Real-World Applications

The CD audio standard (44.1 kHz) was chosen to capture the full human hearing range (up to ~20 kHz) with a safety margin — a direct application of Nyquist. Medical ECG machines sample at 500–1000 Hz with 16-bit resolution; the anti-aliasing filter removes 60 Hz power line interference before sampling. Industrial motor drives use timer-triggered ADC for current feedback — even microseconds of jitter cause control instability at high loop gains. SAR ADCs dominate microcontroller applications because they balance speed, accuracy, and silicon area. The TIP120 Darlington is used in Lab 9 to drive a DC motor from a 3.3 V GPIO pin.

Group 4 Serial Communication & IoT
›

Why This Matters

No embedded system is an island. Every real product communicates — with sensors over SPI, with a PC over UART, with the internet over Wi-Fi. These protocols are precisely timed bit sequences with well-defined electrical and logical specifications. Understanding them at the register level means you can diagnose any failure with a logic analyzer instead of guessing at library bugs. IoT extends this to the cloud, enabling embedded systems to exchange data with web servers in real time.

Concepts

UART (Universal Asynchronous Receiver/Transmitter) is the simplest serial protocol — no shared clock. A frame consists of a start bit (0), 8 data bits LSB-first, and a stop bit (1). Both sides must agree on baud rate in advance (commonly 9600 or 115200 bps). A mismatch produces garbled output. UART is the primary debug output channel throughout this course via PuTTY on the host PC.

SPI (Serial Peripheral Interface) is synchronous — it uses a shared clock (SCLK), MOSI (data out), MISO (data in), and CS (chip select, active-low). SPI runs much faster than UART; the TM4C123 SSI module runs to 25 MHz. CS must be held low for the entire transaction — toggling it between bytes resets the slave device state machine. SPI drives the ST7735R LCD, the TLV5618 DAC, and the SD card in this course.

The Internet of Things (IoT) connects embedded systems to networks. The ESP8266 Wi-Fi module interfaces to the TM4C123 via UART using AT commands — it joins a Wi-Fi network, opens a TCP socket, sends HTTP or MQTT messages, and receives responses. A client-server architecture separates devices that request data (the embedded system as client) from those that serve it (a weather API or MQTT broker as server). The MQTT protocol is a lightweight publish-subscribe messaging system designed for constrained IoT devices operating over unreliable networks.

Bandwidth is the maximum data rate a channel can carry — it may be limited by the I/O device (I/O-bound) or by the software (CPU-bound). If I/O-bound, faster hardware helps; if CPU-bound, better algorithms or a faster clock help. Latency for IoT adds tens to hundreds of milliseconds from network stack overhead — never assume network responses are instantaneous in a timing-critical system.

Key Takeaways

  • UART baud rate must match on both sides — even a 1% error accumulates across a long frame and misaligns the stop bit
  • SPI CS must remain low for the entire transaction — any glitch resets the slave and corrupts the transfer
  • Use a FIFO queue and UART receive interrupt to avoid dropped characters when the main loop is busy
  • IoT adds non-deterministic network latency — decouple network operations from real-time sensor loops using double buffering
  • AT commands to the ESP8266 are text strings — always implement timeout detection, network errors will occur

Real-World Applications

UART is how virtually every microcontroller communicates with a PC for debugging — from hobby boards to aerospace flight computers. SPI drives the displays in smartwatches and the flash memory in every SSD. MQTT runs on millions of IoT devices — smart thermostats, industrial sensors, connected medical monitors. The ESP8266 chip powers a vast ecosystem of Wi-Fi products from smart plugs to environmental monitors, and its successor the ESP32 is one of the most widely deployed IoT chips ever made.

Group 5 Sound Generation & Digital-to-Analog Conversion
›

Why This Matters

Audio is the most perceptually immediate output an embedded system can produce — you hear a bug the instant it appears. Building a music player forces you to simultaneously understand DAC hardware, interrupt-driven output, lookup tables, SPI timing, and the relationship between sample rate and audio quality. The result is immediately evaluable by ear, making this one of the richest learning experiences in the course. The same waveform generation principles apply to control signal synthesis, arbitrary function generators, and PWM audio.

Concepts

A DAC (Digital-to-Analog Converter) converts a binary code to a proportional analog voltage. The TM4C123 has no internal DAC. Two external approaches are used: an R-2R resistor ladder uses only R and 2R valued resistors and N GPIO pins to produce 2ᴺ output levels via superposition — simple and cheap but limited by GPIO drive strength and resistor matching. The TLV5618 is a 12-bit SPI-interface DAC — higher resolution, lower output impedance, cleaner waveforms at the cost of an SPI transaction per sample.

Sound generation requires writing a new DAC sample at the audio sample rate fₛ (typically 8–44.1 kHz). A SysTick or timer ISR fires at fₛ, reads the next value from a lookup table of precomputed waveform samples stored in ROM, and writes it to the DAC. For note frequency f, the step through a table of length N is: step = (N × f) / fₛ. A larger step = higher pitch. The same table generates any note — elegant and memory efficient.

Key DAC performance metrics: resolution (bits — 12-bit gives 4096 levels, ~0.8 mV/step at 3.3 V), settling time (how fast the output reaches its final value — limits max sample rate), SNR (~6 dB per additional bit — 12-bit gives ~72 dB), and glitches (transient spikes when multiple bits switch simultaneously, common in R-2R).

The DAC output cannot drive a speaker directly — speakers are 8–32 Ω and require hundreds of milliwatts. The TPA731 audio amplifier boosts the DAC output to speaker-driving levels. A series resistor between the amplifier output and speaker limits current and sets volume. The SPI timing protocol for the TLV5618 requires CS asserted low → 16 clock cycles of data MSB-first → CS deasserted → DAC latches the new value.

Key Takeaways

  • Sample rate must exceed 2× the highest audio frequency — for notes to 4 kHz, sample at ≥ 8 kHz minimum
  • Never block inside the DAC ISR — it must write one sample and return in under 1/fₛ seconds or audio glitches
  • Use a volatile global to pass DAC values between ISR and main — never share non-atomic data structures
  • R-2R output is noisy and high-impedance; for clean audio use an external SPI DAC with a reconstruction filter
  • The step size and table length are independent — step controls pitch, table length controls waveform resolution

Real-World Applications

Lookup table DAC synthesis is how video game consoles produced music in the 1980s and how modern synthesizers generate waveforms digitally. Audio DACs in smartphones use the same principle at 24-bit, 192 kHz resolution. Medical infusion pumps use DACs to control motor drive signals with high precision. The TPA731 is a class-AB linear amplifier — the same topology as home stereo systems, scaled to milliwatts. Arbitrary waveform generators used in electronics labs are essentially high-speed DACs with lookup table output.

Group 6 Analog Front-End: Op-Amps, Sensors & Filters
›

Why This Matters

The real world does not produce clean 0–3.3 V signals matched to ADC input ranges. Real sensors produce millivolt outputs buried in noise, at impedance levels that would load a GPIO flat, with common-mode interference from power supplies and radiated fields. The op-amp circuits and analog filters here are the hardware that makes sensors actually usable — they are the difference between a precision measurement system and an expensive noise generator.

Concepts

A transducer converts a physical quantity — temperature, pressure, light, force — into an electrical signal. Common examples: a thermistor whose resistance changes predictably with temperature (used in Lab 8); a photoresistor sensitive to light; a piezoelectric sensor producing voltage proportional to force. Outputs are typically small (millivolts to hundreds of millivolts) and require amplification and filtering before digitization.

An operational amplifier (op-amp) is a high-gain differential amplifier used in closed-loop configurations. Key configurations: non-inverting (gain = 1 + Rf/R1, infinite input impedance — ideal for buffering high-impedance sensors); inverting (gain = −Rf/Rin); difference amplifier (amplifies Vp − Vn while rejecting common-mode). Rail-to-rail op-amps (OPA2350, AD8032) are required with a 3.3 V single supply — standard op-amps cannot swing near the supply rails.

An instrumentation amplifier (INA) is a precision differential amplifier with high input impedance and gain set by a single external resistor. The INA122P: G = 5 + 80 kΩ/R_G. Its high CMRR (Common-Mode Rejection Ratio) suppresses noise appearing equally on both inputs — critical when sensor leads pick up interference over long wire runs.

Analog filters shape frequency content before digitization. A first-order RC LPF gives −20 dB/decade rolloff — too gentle for most anti-aliasing. A 2-pole Butterworth LPF gives −40 dB/decade with a maximally flat passband (no in-band ripple). Cutoff frequency fc = 1/(2π√(R1R2C1C2)). In Lab 8, the lpf.xls spreadsheet designs the anti-aliasing filter for the thermistor measurement chain. A highpass filter removes DC offset and low-frequency drift from AC-coupled sensor signals.

Threshold detection uses a comparator to produce a digital output when an analog input crosses a reference voltage — converts sensor signals into GPIO-compatible digital events. Input capture records the exact timer count at each GPIO edge, enabling precise period and frequency measurement of encoder or tachometer pulses.

Key Takeaways

  • Match the sensor range to the ADC input range — a 0–100 mV sensor into a 3.3 V ADC wastes 97% of resolution
  • Op-amp input impedance must greatly exceed source impedance — loading a high-impedance sensor distorts the reading
  • INA122P gain resistor must be 1% tolerance — gain error directly becomes measurement error
  • A 2-pole Butterworth filter has the flattest passband — no ripple, ideal when in-band accuracy matters
  • Rail-to-rail op-amps are mandatory with single 3.3 V supply — standard op-amps cannot output near the rails

Real-World Applications

Active lowpass filters are in every data acquisition system from seismometers (fc ~ 100 Hz) to studio audio interfaces (fc ~ 20 kHz). The thermistor circuit in Lab 8 is inside HVAC systems, medical incubators, and industrial ovens. Input capture timers measure rotation speed in every automotive engine, industrial servo, and hard disk drive.

Why This Matters

Breadboards are for prototyping. Every real product runs on a PCB — and PCB fabrication has a hard, non-negotiable deadline. A late Gerber file means you pay for your own board. This group bridges the gap between a working breadboard prototype and a real engineered product, covering everything from writing a requirements document to calculating battery life to routing copper traces.

Concepts

Content from Lec 29 (Power Supplies/Regulators), Lec 30 (Specifications), Lec 32 (SD Cards, File Systems), Lec 33 (File Systems FAT16), Lec 34 (Enclosures/Connectors) will be added once uploaded. The following is from Lec 26 and Lec 31.

A PCB (Printed Circuit Board) replaces point-to-point wiring with etched copper traces on an insulating substrate. Construction (2-layer): top copper (signal routing), FR4 substrate, bottom copper (ground/power). Trace width determines current capacity — 8 mil for digital signals, 20 mil for power rails. Fabrication specs for this course: minimum trace width 6 mil, minimum clearance 6 mil, minimum drill 15 mil, max board 60 in².

PCB layout rules from Lec 26: place all fixed-location parts first (connectors on edges); place bypass caps as close as possible to each IC power pin; avoid 90° trace corners (use 45° instead — reduces acid traps during etching and looks better); separate analog and digital grounds; add test points at strategic locations; run ground first, then power, then critical signals.

A requirements document (Lec 31) states what the system will do — not how. It is unambiguous, complete, verifiable, and modifiable. Sections: Overview (objectives, roles, interactions with existing systems), Function Description (functionality, performance, usability), Deliverables. It serves as a legally binding agreement between the engineer and the client.

Power budget analysis: battery capacity in mA-Hour = sustained current × hours. An AA battery (1.5 V, 2890 mAh) stores ≈ 15,600 J. Average current = (I_active × duty_active) + (I_sleep × duty_sleep). Battery life = capacity / average current. Li-ion (3.7 V, 2600 mAh) with LP2950 LDO regulator (400 mV dropout) is the standard Lab 7/10 power supply.

Key Takeaways

  • Trace width is a current-carrying constraint: 8 mil for digital, 20 mil for power — use the IPC-2221 table
  • Bypass capacitors must be as close as possible to each IC power pin — long traces defeat their purpose
  • A requirements document states WHAT the system does, never HOW — mixing them creates unmaintainable specs
  • Battery life = capacity (mAh) / average current (mA) — calculate weighted average across all operating modes
  • PCB files are due by a hard deadline — late submission means paying for your own fabrication
  • Route ground first, then power, then critical signals — ground quality affects everything else

Real-World Applications

Requirements documents are how all commercial embedded products are specified — from FDA-regulated medical devices (where the requirements document is a legal submission) to automotive ECUs (ISO 26262 mandates formal requirements traceability). Battery budget calculations determine whether a product ships with a coin cell or a lithium pack — and getting it wrong means a product recall. PCB layout is how every chip in your phone, laptop, and car connects to the rest of the system.

Group 8 Motors & Actuators
›

Why This Matters

Motors convert electrical energy into mechanical motion — they are in every robot joint, EV drivetrain, HVAC fan, drone, hard drive, and industrial machine. Controlling them precisely requires understanding both the physics (back-EMF, inductance, torque) and the electronics (driver circuits, PWM, current sensing). Getting this wrong destroys hardware — a motor without a flyback diode will take out its driver transistor on the very first switch-off.

Concepts

DC motors convert electrical power to rotation. Torque is proportional to current; speed is proportional to voltage minus back-EMF. Back-EMF is the voltage generated by the spinning motor acting as a generator — it opposes applied voltage and grows with speed. At stall (speed = 0), back-EMF = 0 and current is maximum (Vsupply / Rwinding). This is why motors draw the most current at startup.

PWM (Pulse-Width Modulation) controls average motor power by varying the fraction of time (duty cycle) the drive signal is high. At 50% duty cycle, average voltage = 0.5 × Vsupply. The TM4C123 timer generates PWM at programmable frequency and duty cycle. The TIP120 Darlington (hfe ≥ 1000) acts as the power switch — a 3 mA GPIO drives 3 A of motor current. A flyback diode across the motor clamps the back-EMF spike on every PWM turn-off.

Speed measurement uses input capture: a timer records the exact time of each tachometer pulse. RPM = 60 / (pulse_period × pulses_per_rev). The TM4C123 timer input capture mode achieves sub-microsecond resolution.

Stepper motors move in discrete angular steps (typically 1.8° = 200 steps/rev) by energizing coils sequentially — open-loop position control without an encoder. Used in 3D printers, CNC machines, and camera actuators. Servo motors accept a PWM command (1–2 ms pulse width) and hold the commanded angle using an internal position loop. BLDC (Brushless DC) motors replace mechanical brushes with electronic commutation — higher efficiency and longer life; used in drones, EVs, and computer fans.

Key Takeaways

  • PWM frequency should exceed 20 kHz for quiet motor operation — audible whine occurs at lower frequencies
  • Always place a flyback diode across the motor — back-EMF spikes occur every PWM cycle and will destroy the driver transistor
  • Stall current = Vsupply / Rwinding — can be 10× running current; always size the driver for stall conditions
  • Stepper motors lose steps if commanded faster than load torque allows — use acceleration ramps for large inertia loads
  • Input capture gives sub-microsecond speed measurement resolution at 80 MHz timer clock

Real-World Applications

PWM motor control is used in every electric vehicle (Tesla uses three-phase PWM inverters at 10–20 kHz), every drone (ESCs with BLDC motors), and every computer fan. Stepper motors position print heads in inkjet printers and extruders in every 3D printer. Servo motors actuate every joint in industrial robot arms. The PI motor controller in Lab 9 is structurally identical to field-oriented control in EV drives — just at lower power.

Fraction of PWM period the signal is high; 0% = off, 100% = full on; controls average motor voltage

Group 9 Control Systems
›

Why This Matters

A motor without feedback is open-loop — it goes where commanded but has no way to self-correct for load changes, friction, or component variation. Closed-loop control continuously measures the actual output and adjusts the drive to minimize error. Cruise control holding speed on a hill, a drone hovering in wind, a hard drive head landing on track — all are closed-loop systems. The PI controller in Lab 9 is the same mathematical structure used in industrial servo drives running hundred-kilowatt machinery.

Concepts

A feedback control system measures the actual output (process variable), compares it to the desired value (setpoint), computes an error, and adjusts the plant input to reduce the error. The plant is the physical system being controlled — a DC motor with its mechanical load.

A proportional (P) controller computes: output = Kp × error. It responds immediately to error but exhibits steady-state error — a persistent offset that cannot be eliminated because at steady state, error → 0 means output → 0, which cannot overcome friction or load torque. Increasing Kp reduces steady-state error but eventually causes oscillation.

A PI (Proportional-Integral) controller adds: output = Kp × error + Ki × Σerror. The integral accumulates past errors — if any error persists at steady state, the integral grows until the output is large enough to eliminate it, achieving zero steady-state error. In discrete time on the TM4C123: output[n] = Kp × e[n] + Ki × sum(e[0..n]), computed every 10 ms in a SysTick ISR. Excessive Ki causes integral windup and oscillation; clamp the integral when the output saturates (anti-windup).

Tuning approach: increase Kp until the motor tracks the setpoint quickly without oscillating. Then add Ki to remove the remaining steady-state offset. Increase Ki slowly — too much causes oscillation that Kp alone cannot damp. The control loop must run at a rate at least 10× faster than the plant bandwidth for stability.

Key Takeaways

  • P-only control always has steady-state error — integral action is required to achieve zero offset
  • Run the control loop at a fixed rate via timer interrupt — variable timing at high gains causes instability
  • Anti-windup is essential: stop accumulating the integral when output saturates at 0% or 100% duty cycle
  • Tune Kp first for speed, then Ki for accuracy — adding both simultaneously makes instability diagnosis impossible
  • The control loop must run at least 10× faster than the plant bandwidth — for a DC motor this means > 100 Hz

Real-World Applications

PI controllers are the most widely deployed control algorithm in the world — over 90% of industrial control loops use PID or PI. Every thermostat, cruise control, robot joint, and motor drive uses feedback control. The anti-lock braking system in your car runs a PI loop on each wheel at hundreds of Hz. Autopilots run cascaded PI loops for altitude, speed, and attitude. The voltage regulator on your laptop motherboard runs a PI loop at MHz switching frequencies. This material is prerequisite to ECE 445M (Real-Time Operating Systems).

Group 10 Communication Theory & FFT
›

Why This Matters

Every wireless system — Wi-Fi, Bluetooth, LTE, GPS, radar — is built on signal processing mathematics. The FFT is the algorithm that makes real-time spectrum analysis possible on embedded hardware. It lets you see the frequency content of a signal without an expensive spectrum analyzer — which frequencies are present, what the noise floor is, and whether harmonics indicate distortion. Sound input, frequency analysis, and digital communication are different faces of the same underlying theory.

Concepts

The Discrete Fourier Transform (DFT) converts N time-domain samples into N frequency-domain coefficients. The kth coefficient corresponds to frequency k × fₛ/N Hz and represents the signal amplitude and phase at that frequency. A 16-point DFT at 1600 Hz gives bins at 0, 100, 200, ... 1500 Hz. The DFT reveals which frequencies are present — essential for audio processing, channel characterization, and fault detection in mechanical systems.

The Fast Fourier Transform (FFT) computes the DFT in O(N log N) operations instead of O(N²) — for N = 1024 this is a 100× speedup. The FFT exploits symmetry in the DFT butterfly structure to eliminate redundant multiplications. A 16-point FFT runs in real time on the TM4C123. The provided FFT16.xls tool visualizes FFT magnitude for lab use.

Sound input digitizes a microphone signal with the TM4C123 ADC. The microphone output is amplified, DC-biased to ADC midrange (1.65 V for 3.3 V reference), and anti-alias filtered before sampling at fₛ. Applying the FFT to blocks of N samples produces a live frequency spectrum on the LCD display — the same principle as a guitar tuner, spectrum analyzer, or voice recognition frontend.

Communication theory: information is encoded onto signals and transmitted despite noise. Key metrics: bandwidth (frequency range of signal or channel), SNR = 10 log₁₀(P_signal/P_noise) in dB, and modulation (encoding data by varying amplitude AM, frequency FM, or phase PM of a carrier). Shannon's capacity theorem gives the theoretical maximum data rate: C = B log₂(1 + SNR) bits/s for bandwidth B and SNR.

Key Takeaways

  • DFT bin k = frequency k × fₛ/N Hz — for 16-point DFT at 1600 Hz: bin 1 = 100 Hz, bin 8 = 800 Hz
  • Apply a window function (Hanning, Hamming) before the FFT to reduce spectral leakage from signal discontinuities at block edges
  • FFT magnitude = √(real² + imag²) — phase is often discarded for spectrum display applications
  • Averaging multiple FFT frames reduces the noise floor by 3 dB for each doubling of averages
  • Frequencies above fₛ/2 are mirrors of the lower bins — always anti-alias filter before ADC input

Real-World Applications

The FFT is one of the most important algorithms in engineering — it enables MP3 audio compression (MDCT), OFDM modulation in Wi-Fi and 5G, radar signal processing, seismic analysis, and MRI image reconstruction. Guitar tuner apps run an FFT on the microphone input to identify the dominant frequency. The same 16-bin display you build for the LCD runs on supercomputers analyzing gravitational wave signals from the LIGO observatory — same mathematics, vastly different scale.

Group 11 Real-Time Systems
›

Why This Matters

An embedded system that gives the right answer too late is wrong. Real-time does not mean fast — it means predictable. A pacemaker that fires 10 ms late is a medical emergency. An anti-lock brake system that responds 100 ms late causes a crash. A motor control loop that misses a deadline allows instability to grow. This distinction between real-time and merely fast is what separates firmware that works reliably in the field from firmware that only works in the lab under ideal conditions.

Concepts

A real-time system is one where correctness depends both on logical results and the time at which they are produced. Three categories: hard real-time — missing any deadline is catastrophic (pacemakers, airbags, flight control); firm real-time — occasional misses degrade quality but are not catastrophic (industrial control); soft real-time — misses degrade quality gracefully (video streaming, user interfaces).

A periodic task must execute once every T seconds and complete within deadline D ≤ T. Jitter is the variation in actual execution timing from the ideal — in hard real-time systems it must be bounded: −k ≤ δᵢ ≤ +k for all i. Jitter sources: interrupt latency, competing ISRs, variable-length instructions. Reduction: assign the task to a high-priority timer ISR. Elimination: use hardware-triggered ADC conversion that bypasses all software latency.

A race condition occurs when system behavior depends on the relative timing of concurrent events — a non-deterministic bug that appears and disappears based on timing. Race conditions on shared data produce data hazards: RAW (read-after-write), WAW (write-after-write), WAR (write-after-read). All require atomic access to prevent corruption — through interrupt disabling, hardware atomic operations, or lock-free data structures.

The simplest embedded scheduler is a foreground/background system: a main loop (background) interrupted by ISRs (foreground). This course uses this model throughout. Rate-monotonic scheduling is optimal for fixed-period preemptive tasks: assign highest priority to the shortest period. A set of N tasks is schedulable when total CPU utilization U = Σ(Cᵢ/Tᵢ) ≤ N(2^(1/N)−1), approaching ln 2 ≈ 0.693 for large N. Engineering ethics: software failures in safety-critical real-time systems have killed people — the Therac-25 race condition, the Boeing 737 MAX MCAS software. Engineers have an ethical obligation to understand failure modes, test corner cases, and communicate limitations honestly.

Key Takeaways

  • Real-time means predictable, not fast — a system responding in exactly 10 ms every time is more real-time than one averaging 1 ms with outliers
  • Jitter must be bounded in hard real-time — use hardware timer-triggered tasks, not software polling loops
  • Race conditions are non-deterministic — they may pass all lab testing and only appear in the field under specific timing
  • Rate-monotonic scheduling: highest priority to shortest period — provably optimal for fixed-period preemptive tasks
  • Keep CPU utilization below ~70% — hard real-time systems need margin for worst-case interrupt storms

Real-World Applications

Real-time systems theory governs every safety-critical embedded design: avionics flight controllers (DO-178C requires formal deadline analysis), automotive ECUs (AUTOSAR defines task scheduling), and FDA-regulated medical devices (formal software hazard analysis required). The foreground/background model used in this course is identical to what runs in simple consumer products — appliance controllers, power tools, basic sensors. FreeRTOS, VxWorks, and QNX implement rate-monotonic scheduling for complex products. This material is the foundation for ECE 445M.
