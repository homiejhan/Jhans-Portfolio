Group 1 Intro to Embedded Systems & ARM Architecture
›

Why This Matters

Every device around you with a microcontroller — your car's engine management unit, a medical infusion pump, a drone flight controller, an industrial robot — was built by engineers who understood both the hardware and the software. ECE 445L exists because that dual fluency is exactly what industry pays a premium for. As McDermott puts it: software-centric students need to become proficient with hardware, and hardware-centric students need to become proficient with software. This first group gives you the foundation everything else is built on.

Concepts

An embedded system is a computer built to perform a dedicated function within a larger device. Unlike a general-purpose PC, it operates under tight constraints — limited memory, fixed power budget, and real-time deadlines. The course hardware is the Texas Instruments TM4C123 LaunchPad, a development board built around the ARM Cortex-M4 — the most widely deployed 32-bit processor architecture on Earth.

The Cortex-M4 is a RISC (Reduced Instruction Set Computer) with a Harvard architecture: separate buses for instructions and data allow simultaneous fetch and memory access. Key features: few opcodes, fixed-length instructions, many general-purpose registers (R0–R15), and a 3-stage pipeline. R13 is the stack pointer, R14 the link register (return address), and R15 the program counter.

The TM4C123 uses memory-mapped I/O — every peripheral register (GPIO, timer, ADC, UART) is assigned a fixed address in the 4 GB address space. Reading and writing peripherals is identical to reading and writing RAM, just at different addresses. The memory map: Flash at 0x0000.0000 (256 KiB), SRAM at 0x2000.0000 (32 KiB).

Fixed-point arithmetic is essential when floating-point is too slow or unavailable. A fixed-point number is represented as an integer I and a fixed constant Δ (the resolution), where the true value \(= I \cdot \Delta\). Key design choices: signed vs. unsigned, number of bits (8/16/32), and the resolution Δ. Range is the number of distinguishable values (determined by bit width); resolution is the smallest detectable step (equal to Δ). A practical example: a 12-bit ADC with Vref = 3.3 V gives Δ = 3.3/4096 \(\approx\) 0.000806 V/bit. Storing millivolts as an integer (\(\Delta = 0.001\,\text{V}\)): \(I = (3300 \times N) \gg 12\). Watch for overflow (promoting to 32-bit before multiply) and dropout (underflow from shifting before multiplying).

ESD (Electrostatic Discharge) can instantly destroy a LaunchPad — the human body can accumulate 3,000–5,000 V. Always handle boards by the edges and ground yourself first.

Key Takeaways

  • The Cortex-M4 is RISC/Harvard: few opcodes, fixed-length, separate instruction and data buses, 3-stage pipeline
  • All peripheral control is through memory-mapped registers — the datasheet is the source of truth for every address and bit field
  • Fixed-point value \(= I \cdot \Delta\): choose Δ based on the resolution you need, choose bit width based on the range you need
  • Overflow and dropout are the two failure modes of fixed-point arithmetic — always cast to uint32_t before multiplying large values
  • Accuracy (closeness to true value), precision (reproducibility), and resolution (smallest detectable change) are three distinct concepts

Real-World Applications

Fixed-point arithmetic is how virtually every real-time DSP system on embedded hardware works — from the audio codec in your phone to the motor controller in an electric vehicle. The ARM Cortex-M4 is the processor inside products from hearing aids (Cochlear) to automotive ECUs (Bosch) to satellites. Understanding the architecture at the register level is what allows you to debug hardware-software interactions that no simulator can reproduce.

Group 2 Interrupts, Debugging, I/O Synchronization & Critical Sections
›

Why This Matters

Real embedded systems have to respond to multiple asynchronous events simultaneously — a button press, a timer firing, a byte arriving over serial. The processor runs at MHz; peripherals run at kHz or Hz; they are completely asynchronous with each other. Without a systematic approach to synchronizing these mismatched speeds, data gets lost, events get missed, and the system behaves unpredictably. This group covers the full spectrum: from the simplest possible approach (just wait and hope) to the most capable (interrupt-driven FIFO queues), and gives you the debugging tools to observe what the system is actually doing.

Concepts

I/O Synchronization. Every I/O device has two states: Busy (not ready) and Done (ready for service). For an input device, the transition from Busy to Done means new data has arrived; for an output device it means the device is ready for new data. The fundamental challenge is: the processor and the I/O device operate on completely different time scales, and the processor must coordinate correctly with both.

There are four synchronization strategies. Blind-cycle: the software simply waits a fixed, predetermined time, then reads or writes — no status checking. Simple and predictable, but wastes CPU time and fails if the device timing varies. Busy-wait (polling): the software continuously reads the device status register in a loop until it transitions to Done, then immediately services it. Zero latency once the device is ready, but the CPU is 100% occupied waiting — it can do nothing else. Busy-wait is appropriate only when the device timing is predictable, the I/O is simple, there is a fixed load, and there is truly nothing else the system needs to do. Interrupt-driven I/O: the CPU resumes its main work; when the device transitions to Done, it asserts an interrupt line, and the NVIC automatically transfers execution to the ISR. The ISR services the device, then returns. This frees the CPU entirely from polling. For buffered I/O, the ISR writes data into a FIFO queue (input) or reads data from one (output), decoupling the ISR's production rate from the main loop's consumption rate. DMA (Direct Memory Access): a dedicated hardware engine transfers data directly between memory and peripherals without any CPU involvement — lowest latency, highest bandwidth. Used for high-rate streams like continuous DAC audio output.

Threads and the NVIC. A thread is the path of execution of software. When an interrupt fires, a new background thread (the ISR) is created alongside the main foreground thread. All threads share global variables but have their own local variables on the stack. The NVIC manages 77 interrupt sources on the TM4C123 across 8 priority levels — a higher-priority ISR can preempt a lower-priority one. Three things are always required: arm the device (set its arm bit so the flag can trigger an interrupt), enable the interrupt in the NVIC with a priority, and set the global enable (I=0 via CPSIE). The PRIMASK bit disables all interrupts globally (I=1 postpones, not dismisses). The hardware sets a flag when requesting service; the ISR must clear it (acknowledge) before returning or the interrupt fires immediately again.

Edge-triggered interrupts and debouncing. Edge-triggered interrupts fire on a GPIO voltage transition — rising, falling, or both. On the TM4C123 there are 77 interrupt vectors; GPIO Port F is vector 46 (IRQ 30). When Port F fires: the current instruction finishes, 8 registers are pushed to the stack, the PC loads the vector address, IPSR is set to 46, and LR is set to 0xFFFFFFF9. Switch debouncing is essential — mechanical contacts bounce 1–20 ms on each press, generating dozens of spurious edges. Software debounce: on first edge, disarm GPIO and start a one-shot timer (~10 ms); when timer fires, re-read switch state and re-arm. Hardware debounce: RC filter (100 kΩ × 0.1 µF \(\approx\) 10 ms).

FIFO Queues. A FIFO (First-In First-Out) queue decouples the producer (ISR writing data) from the consumer (main loop reading data). Let tₚ be the average time to produce one item and tᶜ the average time to consume one. The system is stable only if tₚ > tᶜ — the consumer must be faster than the producer on average, or the FIFO will always fill. If the FIFO becomes full and tₚ > tᶜ, increasing the FIFO size helps. A full error on put is catastrophic (data lost); an empty error on get is benign (just wait). Little's Theorem relates queue depth to throughput: \(N = \lambda R\), where N is the average number of items in the system, λ is the average arrival rate, and R is the average response time. The system is stable when \(\lambda < C\) = 1/S (arrival rate less than service rate).

Latency and jitter. Latency is the time between when an event occurs and when the ISR responds. For periodic events (ADC, DAC, control loops), latency variation is jitter. With one ISR, jitter spans only ±2–3 clock cycles. With multiple competing ISRs, jitter grows to thousands of cycles. Elimination: use hardware timer-triggered ADC — the timer directly triggers the ADC conversion, bypassing all software latency.

Critical sections. A critical section exists when: (1) a global variable is shared between threads, (2) the access is non-atomic, and (3) at least one thread writes. Solutions: disable interrupts around the shared access (make atomic), or restructure data so the access is a single 32-bit load or store (e.g., pack Hour and Minute into one uint32_t). A function is reentrant if it can be called simultaneously from two threads without corruption — achieved with local variables only, or atomic global access.

Debugging. Two types: performance debugging (timing — is the system meeting deadlines?) and functional debugging (data — is the computation correct?). Intrusiveness is the degree to which the debugger perturbs the system. Non-intrusive: logic analyzers, oscilloscopes. Minimally intrusive: dump arrays, JTAG, assertions. Highly intrusive: printf, breakpoints, single-stepping. In this course: any time you would use printf, use a dump array instead — save data to a global array and inspect it post-mortem in the debugger. A heartbeat GPIO toggle confirms the program is running and can be captured on a logic analyzer with zero CPU overhead.

Key Takeaways

  • Three steps for any interrupt: arm → enable in NVIC with priority → global enable (I=0); miss any one and the interrupt silently fails
  • Busy-wait is only appropriate when there is truly nothing else to do — it consumes 100% of the CPU
  • The ISR must acknowledge (clear the flag) before returning — failure causes an infinite re-interrupt loop
  • FIFO stability requires arrival rate < service rate (tₚ > tᶜ) — if the consumer is slower than the producer, no amount of buffering helps
  • A full FIFO means lost data — this is catastrophic; an empty FIFO just means the consumer waits
  • In this class, any time you want to debug with printf, use a dump and observe the data later — printf is too intrusive for real-time systems
  • Switch bounce is real — always debounce, or a single button press generates dozens of interrupt events

Real-World Applications

The four synchronization strategies map directly to real products: blind-cycle is used in simple timing delays and initialization sequences. Busy-wait is used in bit-banged SPI on ultra-low-power microcontrollers where the CPU has nothing else to do during the transfer. Interrupt-driven FIFO queues are how every UART, USB, and Ethernet driver works — data arrives asynchronously and is buffered for the application. DMA is how audio cards stream samples to DACs at 192 kHz without burning any CPU cycles. Little's Theorem is the foundational result used by network engineers to size buffers in routers and switches — it tells you how big the queue needs to be to achieve a target response time at a given traffic load.

Group 3 Data Acquisition Systems

Why This Matters

The physical world is analog. Sensors produce voltages; motors need voltages. The ADC and DAC are the hardware bridges between the digital processor and reality — and their characteristics directly determine what your system can measure and control. Equally important: if you sample incorrectly, you corrupt your data in a way that is mathematically irreversible. Every instrument from a handheld multimeter to a medical ECG to an industrial oscilloscope is fundamentally a data acquisition system.

Concepts

The Nyquist-Shannon Sampling Theorem states: a band-limited signal with no frequency components above W Hz may be completely recovered from samples taken at a rate greater than 2W samples per second. In practice, Valvano's Postulate adds a stricter engineering rule: sample at more than \(10\times\) the highest frequency of interest for the reconstructed waveform to visually resemble the original. An anti-aliasing analog lowpass filter must be placed before the ADC to remove all signal content above \(f_s/2\) — content above this frequency folds back into the baseband as aliasing, which is irreversible. A 100 Hz signal sampled at 1600 Hz reconstructs perfectly; the same signal sampled at just above its Nyquist rate looks nothing like the original in the time domain even though it mathematically satisfies the theorem.

A complete Data Acquisition System (DAS) consists of: measurand → transducer → analog preamplifier → analog filter → sample-and-hold → ADC → microcomputer → actuator. Each stage introduces its own errors. Key quantitative DAS parameters: range (rₓ — total span of measurable values), resolution (Δx — smallest detectable change), precision (number of distinguishable alternatives nₓ), repeatability (σ of repeated measurements under same conditions), and reproducibility (σ under different conditions). Qualitative parameters for detection systems: true positive (event occurs, system detects it), false positive (event doesn't occur, system reports it), false negative (event occurs, system misses it), true negative (event doesn't occur, system correctly says so). Sensitivity = TP/(TP+FN); Specificity = TN/(TN+FP).

Three ADC architectures: A Successive Approximation (SAR) ADC performs a binary search — each bit is decided by comparing VIN to a DAC output, retaining or rejecting it. Example: converting 45 (6-bit SAR): Is X≥32? Yes (retain) → 0x1. Is X≥48? No (reject) → 0x10. Is X≥40? Yes → 0x101. Is X≥44? Yes → 0x1011. Is X≥46? No → 0x10110. Is X≥45? Yes → 0x101101 = 45. The TM4C123 uses a 12-bit SAR at up to 1 MSPS — good accuracy, moderate speed. A Flash ADC uses \(2^N - 1\) simultaneous comparators — one clock cycle conversion, extremely fast but area-exponential. A Sigma-Delta ADC oversamples at very high rates with 1-bit quantization then applies a digital decimation filter — excellent resolution for audio (high resolution, low speed), poor for fast distance measurement.

Time jitter in sampling violates the Nyquist theorem (which assumes uniform intervals), introduces errors in derivative computation (dx/dt = (x(t)−x(t−Δt))/Δt), distorts DAC-generated waveforms, and causes instability in control loops. With one interrupt source, jitter spans only ±2–3 clock cycles. With multiple competing interrupts, jitter grows to ±thousands of cycles. Elimination: use timer-triggered ADC sampling (the timer hardware directly triggers the ADC conversion, bypassing software latency entirely).

From Lec 13 on transistor interfaces: BJT operation — in saturation, the transistor acts as a closed switch (Vbe > Vbe(SAT), typically 0.6–1.2 V). Collector current Ic = hfe × Ib. The TIP120 Darlington transistor cascades two BJTs for \(h_{fe} \geq 1000\), allowing a 3 mA GPIO signal to control 3 A of motor current. MOSFET operation: voltage-controlled current source; on when Vgs > threshold; Rds \(\approx\) 0 when on, Rds = ∞ when off. Digital logic interfacing requires VOH > VIH, IOH > IIH, VOL < VIL, IOL > IIL — the TM4C123 GPIO can drive 2/4/8 mA (configurable). To interface to the ESP8266 (Vih = 2.475 V, Voh(TM4C) = 2.4 V), a level shifter or pull-up resistor is required since Voh barely meets Vih. Inductive loads (motors, solenoids) require a flyback diode across the load to clamp the back-EMF voltage spike when switching off — without it, the spike can permanently destroy the driver transistor.

Temperature Data Acquisition Example (Lec 40). A complete thermistor DAS illustrates every stage: Specifications: range 10–40 °C, frequencies 0–10 Hz, resolution ≥ 0.1 °C. Design steps: (1) Transducer: NTC thermistor (30 kΩ at 25 °C, \(R_T = R_0 e^{\beta/T}\)); perform 2-point calibration with a multimeter reference to extract R₀ and β. Good transducer properties: operates in range, fast response, high sensitivity (dR/dT), good selectivity (only responds to temperature), stability over time, interchangeability, linearity (monotonic), and robustness. (2) Bridge circuit: V₁ = +2.50 V from LM4041CILPR precision reference (Lec 35 voltage reference). Thermistor R_T and series resistor R₁ form a half-bridge. Choose R₁ to limit self-heating: P = V²/R < 2.5 mW/°C; set R₂ = R₁ and R₃ = minimum R_T (at max temperature). (3) INA122P instrumentation amp: single-supply, rail-to-rail, G = 5 + 200 kΩ/R_G; choose R_G to map the bridge differential output over 10–40 °C to 0–3.3 V; V_ref input sets the output offset. (4) 2-pole Butterworth LPF at ½ f_s (e.g., 5 Hz cutoff for 10 Hz sampling): with R₁=R₂ and C₁=C₂, f_c = 1/(2πR₁C₁); select C0G capacitors for precision. (5) TM4C123 12-bit ADC: 4096 counts for 0–3.3 V, giving 0.8 mV/count. (6) Software calibration: store (ADC, Temp) pairs in a lookup table; interpolate between entries. ADC reference: LM4041CILPR (bypassed with 0.1 µF ceramic decoupling). SNR determines resolution: add LPF and reduce EM field pickup to improve. Calibration drift determines accuracy: calibrate at use temperature, in the same configuration. Data flow: Thermistor → Bridge → INA122P → LPF → TM4C123 ADC → FIFO → main → LCD display; ADC triggered by Timer0A ISR at fixed rate (avoids jitter); Timer0A also triggers LCD update via FIFO.

Key Takeaways

  • Aliasing is irreversible — the anti-aliasing filter must come before the ADC, not after
  • Nyquist says sample \(> 2W\); Valvano's practical rule says sample >\(10 f_{max}\) for the waveform to look right on a plot
  • SAR ADC is good for accuracy; Flash ADC is fastest; Sigma-Delta is best for high-resolution audio
  • Timer-triggered ADC completely eliminates sampling jitter — software-triggered sampling cannot
  • Accuracy ≠ precision ≠ resolution — a precise ADC can be inaccurate; a high-resolution ADC can be imprecise
  • Always use a flyback diode across any inductive load — the back-EMF spike will destroy the driver without it
  • TM4C123 to ESP8266 interface requires a level shifter — Voh(TM4C) = 2.4 V barely meets Vih(ESP) = 2.475 V

Real-World Applications

The CD audio standard (44.1 kHz) was chosen to capture the full human hearing range (up to ~20 kHz) with a safety margin — a direct application of Nyquist. Medical ECG machines sample at 500–1000 Hz with 16-bit resolution; the anti-aliasing filter removes 60 Hz power line interference before sampling. Industrial motor drives use timer-triggered ADC for current feedback — even microseconds of jitter cause control instability at high loop gains. SAR ADCs dominate microcontroller applications because they balance speed, accuracy, and silicon area. The TIP120 Darlington is used in Lab 9 to drive a DC motor from a 3.3 V GPIO pin.

Group 4 Serial Communication & IoT

Why This Matters

No embedded system is an island. Every real product communicates — with sensors over SPI, with a PC over UART, with the internet over Wi-Fi. These protocols are precisely timed bit sequences with well-defined electrical and logical specifications. Understanding them at the register level means you can diagnose any failure with a logic analyzer instead of guessing at library bugs. IoT extends this to the cloud, enabling embedded systems to exchange data with web servers in real time.

Concepts

UART (Universal Asynchronous Receiver/Transmitter) is the simplest serial protocol — no shared clock. A frame consists of a start bit (0), 8 data bits LSB-first, and a stop bit (1). Both sides must agree on baud rate in advance (commonly 9600 or 115200 bps). A mismatch produces garbled output. UART is the primary debug output channel throughout this course via PuTTY on the host PC.

SPI (Serial Peripheral Interface) is synchronous — it uses a shared clock (SCLK), MOSI (data out), MISO (data in), and CS (chip select, active-low). SPI runs much faster than UART; the TM4C123 SSI module runs to 25 MHz. CS must be held low for the entire transaction — toggling it between bytes resets the slave device state machine. SPI drives the ST7735R LCD, the TLV5618 DAC, and the SD card in this course.

The Internet of Things (IoT) connects embedded systems to networks. The ESP8266 Wi-Fi module interfaces to the TM4C123 via UART5 — it joins a Wi-Fi access point, opens a TCP connection to an MQTT broker, and exchanges publish/subscribe messages. All communication travels through a layered protocol stack: the TCP/IP model maps the OSI 7-layer model into four layers — Application (HTTP, MQTT, SMTP), Transport (TCP or UDP), Internet (IP, ICMP, IGMP), and Media Access/Physical (Ethernet, Wi-Fi). Each layer wraps the layer above it: user data becomes an application message, then a TCP/UDP segment, then an IP datagram, then a physical frame.

TCP (Transmission Control Protocol) is connection-oriented: it performs a handshake, guarantees ordered delivery, retransmits on error or timeout, and uses a 20-byte header. UDP (User Datagram Protocol) is connectionless — an 8-byte header, best-effort delivery, no retransmission, lower latency. TCP is used by HTTP, HTTPS, FTP, SMTP, and MQTT. UDP is used by DNS, DHCP, VOIP, and SNMP. TCP data arrives as a byte stream with no message boundaries; UDP packets have definite boundaries honored upon receipt.

Every device on a network has an IPv4 address — a 32-bit number written as four octets (e.g., 192.168.1.1). Devices on a local intranet use private addresses; a Network Address Translation (NAT) router maps them to a single public IP for internet access. DNS (Domain Name System) resolves human-readable hostnames to IP addresses. A port number identifies the service within a host: port 80 for HTTP, port 443 for HTTPS, port 22 for SSH, port 53 for DNS, port 1883 for MQTT.

The client-server architecture works as follows: the server creates a connection socket (conSocket) at a fixed port and waits for clients. When a client connects using its own clientSocket (providing only the server IP and port), the server spawns a dedicated thread and a unique servSocket for that client — allowing many simultaneous clients. The clientSocket and servSocket form a paired, exclusive channel. When the client closes its socket, the server detects the broken pairing and kills the associated thread. A server can serve zero to hundreds of thousands of clients concurrently this way.

MQTT (MQ Telemetry Transport) is a publisher-subscriber protocol designed for constrained IoT devices over TCP/IP. A central broker receives published messages tagged with a topic string and distributes them to all clients that have subscribed to that topic. There is no direct connection between publisher and subscriber — the broker decouples them. All clients can both publish (broadcast) and subscribe (receive). MQTT brokers do not normally store messages. The MQTT packet structure has a fixed header (1 byte control + 1–4 bytes length), an optional variable-length header, and an optional payload carrying the actual data. MQTT can also run over WebSocket — the client and server start with HTTP, negotiate an upgrade to full-duplex binary WebSocket, then exchange MQTT data over that persistent connection.

Bandwidth is the maximum data rate a channel can carry — it may be limited by the I/O device (I/O-bound) or by the software (CPU-bound). If I/O-bound, faster hardware helps; if CPU-bound, better algorithms or a faster clock help. Latency for IoT adds tens to hundreds of milliseconds from network stack overhead — never assume network responses are instantaneous in a timing-critical system.

IEEE 802.11 (Wi-Fi) Physical Layer. Wi-Fi operates at 2.4 GHz and 5 GHz. Modulation schemes range from ASK, FSK, and PSK up to QAM (Quadrature Amplitude Modulation) — 802.11a/g use 16-QAM and 64-QAM, encoding multiple bits per symbol by independently varying amplitude and phase. 802.11b/g (2.4 GHz) provides 11 channels with 20 MHz bandwidth and 5 MHz spacing — channels 1, 6, and 11 are non-overlapping. Spread spectrum: DSSS (Direct-Sequence Spread Spectrum) in 802.11b; OFDM (Orthogonal Frequency-Division Multiplexing) in 802.11a/g/n — OFDM divides the channel into many narrow orthogonal subcarriers, giving excellent multipath resistance. Network topologies: BSS (Basic Service Set) — up to 2000 stations connect through one AP; IBSS (Independent BSS) — ad-hoc peer-to-peer, no AP; Distribution System — two or more BSSs interconnected. The DAC and ADC quality limits the maximum achievable QAM order, capping data rate.

CC3100 Wi-Fi Module. The CC3100 (TI SimpleLink) is a self-contained Wi-Fi network processor that offloads the entire TCP/IP, TLS/SSL, and Wi-Fi stack from the TM4C123. It interfaces via SPI (SSI2, Port B) and exposes the SimpleLink driver API. Hardware layers inside the CC3100: ARM processor + Crypto Engine + MAC Processor + Baseband + Radio, plus SPI/UART host interface. Software layers on the TM4C123: User Application → SimpleLink Driver → SPI/UART Driver; inside the CC3100: Application Protocols, TLS/SSL, TCP/IP, Supplicant, Wi-Fi Driver, MAC, Baseband, Radio. TCP client flow: sl_Start(0,0,0) → WlanConnect() → sl_NetCfgGet() → DNS lookup sl_NetAppDnsGetHostByName() → sl_Socket(SL_AF_INET, SL_SOCK_STREAM, 0) → sl_Connect() → sl_Send() HTTP GET → sl_Recv() JSON response → sl_Close(). UDP client/server uses SL_SOCK_DGRAM, sl_SendTo()/sl_RecvFrom() — no handshake. The CC3100 SDK includes examples: get_weather, http_client, tcp_socket, udp_socket, mqtt_client, ssl, and more.

Secure Digital Card (SDC) — Flash Memory & SPI Storage. Flash memory uses a stacked-gate NMOS transistor: a Poly1 floating gate stores charge (programming = electrons trapped → high Vt = logic 0; erasing = electrons removed → low Vt = logic 1), and a Poly2 control gate accesses the cell. The oxide stack is Oxide-Nitride-Oxide (ONO) between the gates and a tunnel oxide under the floating gate. Two dominant architectures: NOR flash — each cell has individual drain contact, enabling random byte access at the cost of larger die area; NAND flash — cells are connected in series strings of 16 with no individual contacts, making cells inherently smaller (SD cards use NAND). As lithography shrinks below 30 nm, only ~100 electrons remain on a NAND floating gate — making cells sensitive to charge leakage and reducing endurance. Single-Level Cell (SLC) stores 1 bit/cell (two Vt levels); Multi-Level Cell (MLC) exploits the continuous Vt distribution to store 2 bits/cell (four levels). MLC requires N−1 read reference voltages to distinguish N levels, increasing read complexity and reducing endurance/retention compared to SLC.

The SD card is accessed via SPI using the SSI0 module (TM4C123 Port A: MISO→PA4, SCK→PA2, MOSI→PA5, TFT_CS→PA3). The SD card chip-select (CARD_CS) connects to PB0 as a GPIO — a separate CS from the ST7735 TFT display, allowing both devices to share the same SPI bus. Software is organized into two layers: (1) the low-level driver (diskio.c / diskio.h) provides hardware abstraction — disk_initialize(), disk_status(), disk_read(), disk_write(), and disk_ioctl(); initialization involves SSI0_Init, CS_Init, and a Timer5A_Handler for SD card timing. (2) The high-level FatFs driver (ff.c / ff.h) provides a POSIX-like file API — f_open(), f_read(), f_write(), f_close(), f_lseek(), f_mkdir(), f_mount(), f_printf(), and many more. The FatFs module sits between the user application and diskio — it handles the FAT file system details internally. Use the project SDCFile_4C123 (not SDC_4C123) for file system access; SimpleUnformattedTest for raw sector-level troubleshooting (warning: overwrites formatting). Always format SD cards on a Windows machine, not macOS.

The FAT (File Allocation Table) file system is the universal format for SD cards (FAT16 and FAT32). Three structures work together: (1) the Directory maps each file name to a starting block number; (2) the File Allocation Table — a linked list stored as an array — maps each block to the next block of that file (or a special end-of-chain marker); (3) the Disk contains the actual data blocks. To read File A starting at block 10: look up 10 in the FAT → get block 3 (next) → look up 3 → get block 19 (next) → look up 19 → end of chain. This chained allocation allows files to be fragmented across non-contiguous disk blocks. Free blocks are indicated by a 0 entry in the FAT; reserved sectors (e.g., 0, 1) are marked with a special value (x). File system performance metrics: size (storage efficiency), speed (read/write throughput), and reliability (data integrity across power failures). Integrating the SD card alongside the ST7735 display requires matching the SDC_CS_PB0/SDC_CS_PD7 defines in ST7735.c and diskio.c, and ensuring GPIO_PORTA_PCTL_R, writedata, and writecommand are consistent between the two code bases.

ESP8266 Wi-Fi Module. The ESP8266 (Espressif) is a low-cost Wi-Fi SoC with an 80 MHz processor, 64 KB instruction RAM, 96 KB data RAM, 802.11 b/g/n, and +20 dBm output power. It operates at 3.3 V, draws ~80 mA average (up to 215 mA peak) — requires a dedicated 3.3 V supply. It is controlled from the TM4C123 via UART using AT commands (Hayes-style text commands). Key AT commands and their function: AT+CWMODE=1 (set client mode), AT+CWJAP="ssid","pass" (join AP — returns OK), AT+CIFSR (get assigned IP address), AT+CIPMUX=0 (single connection mode), AT+CWLAP (list available networks), AT+CIPSTART="TCP","host",80 (DNS lookup + open TCP socket — returns OK\r\nLinked), AT+CIPSEND=N then N bytes of data (send + receive; received data prefixed with +IPD,length:), AT+CIPCLOSE (close — returns OK\r\nUnlink). The complete weather server cycle uses openweathermap.org: join AP → CIPSTART → CIPSEND with HTTP GET → parse JSON response → CIPCLOSE.

Key Takeaways

  • UART baud rate must match on both sides — even a 1% error accumulates across a long frame and misaligns the stop bit
  • SPI CS must remain low for the entire transaction — any glitch resets the slave and corrupts the transfer
  • Use a FIFO queue and UART receive interrupt to avoid dropped characters when the main loop is busy
  • IoT adds non-deterministic network latency — decouple network operations from real-time sensor loops using double buffering
  • TCP is reliable but heavyweight (20-byte header, connection setup); UDP is fast but unreliable — choose based on whether your application can tolerate dropped data
  • MQTT brokers decouple publishers from subscribers — the embedded device never needs to know who is consuming its data
  • The CC3100 offloads the entire TCP/IP and Wi-Fi stack — the TM4C123 only calls SimpleLink API functions; all network complexity lives in the module's ROM
  • The ESP8266 uses AT commands over UART — AT+CIPSTART handles DNS + TCP connect in one step; received data arrives prefixed with +IPD,N:
  • IEEE 802.11b/g has 11 channels at 2.4 GHz; channels 1, 6, and 11 are non-overlapping — use these to avoid co-channel interference
  • QAM encodes multiple bits per symbol by varying amplitude and phase simultaneously — 64-QAM encodes 6 bits/symbol but requires very high SNR to decode reliably; DAC/ADC quality limits the maximum achievable QAM order
  • The ESP8266 requires a dedicated 3.3 V supply capable of 215 mA peak — the TM4C123 LaunchPad 3.3 V rail cannot supply this

Real-World Applications

UART is how virtually every microcontroller communicates with a PC for debugging — from hobby boards to aerospace flight computers. SPI drives the displays in smartwatches and the flash memory in every SSD. MQTT runs on millions of IoT devices — smart thermostats, industrial sensors, connected medical monitors. The ESP8266 chip powers a vast ecosystem of Wi-Fi products from smart plugs to environmental monitors, and its successor the ESP32 is one of the most widely deployed IoT chips ever made.

Group 5 Sound Generation & Digital-to-Analog Conversion
›

Why This Matters

Audio is the most perceptually immediate output an embedded system can produce — you hear a bug the instant it appears. Building a music player forces you to simultaneously understand DAC hardware, interrupt-driven output, lookup tables, SPI timing, and the relationship between sample rate and audio quality. The result is immediately evaluable by ear, making this one of the richest learning experiences in the course. The same waveform generation principles apply to control signal synthesis, arbitrary function generators, and PWM audio.

Concepts

A DAC (Digital-to-Analog Converter) converts a binary code to a proportional analog voltage. The TM4C123 has no internal DAC, so external hardware is required. Three main DAC architectures are used: Binary Weighted DAC — one resistor per bit (simple, used in ECE 319K); R-2R Ladder DAC — uses only R and 2R valued resistors in a cascading ladder, every source sees 3R to ground and current divides by 2 at each node, improved precision because only two resistor values are needed; Thermometer (Resistor String) DAC — one resistor per output level (\(2^N\) resistors for N bits), fully monotonic and used in the TLV5616/TLV5618. The TLV5618 output equation: \(V_{out} = \dfrac{2 \cdot REF \cdot n}{4096}\) where n is the 12-bit input code and REF = 1.5 V (giving 0–3 V output range). The TLV5618 has two channels; the TLV5616 is single-channel. Both require the LM4041 adjustable shunt reference to generate a precise 1.5 V reference, set by \(V_Z = V_{REF} \left(1 + \dfrac{R2}{R1}\right)\).

SSI/SPI and the TM4C123. The TM4C123 has four SSI modules (SSI0–SSI3). SSI0 uses PA2–PA5 (drives the ST7735 LCD), SSI1 uses PD0–PD3 (drives the TLV5616/TLV5618 DAC), SSI2 uses PB4–PB7 (drives the CC3100). The SSI module supports data sizes from 4 to 16 bits, Freescale SPI mode (with configurable clock polarity CPOL and phase CPHA), TI synchronous serial mode, and Microwire mode. The TLV5616 SPI timing is Freescale mode: CS (active-low) asserted, then SCLK pulses on the rising edge with DIN data bits from D15 (MSB) to D0 (LSB), then CS deasserted — DAC latches new value. The TM4C123 SSI clock formula: \(f_{SSI} = \dfrac{f_{bus}}{CPSDVR \times (1 + SCR)}\), where CPSDVR is an even number 2–254 and SCR is 0–255. At 80 MHz bus, setting CPSDVR=8, SCR=0 gives 10 MHz SSI clock. The TNF (Transmit FIFO Not Full) bit must be checked before writing to SSI_DR_R — the FIFO is 8 elements deep. Timing analysis: data available (DA) must precede data required (DR) by at least the setup time; data must remain valid past the DR window by at least the hold time. For SSI transmission lines: propagation delay \(t_{pd} = \dfrac{2L}{v_f \cdot c}\) where \(v_f\) = velocity factor (\(\approx 0.6\) for typical PCB traces).

Sound and Music. Human hearing spans approximately 25–20,000 Hz, most sensitive in the 1–5 kHz range. Sound intensity level \(L_I = 10\log_{10}(I/I_0)\) where \(I_0 = 10^{-12}\,\text{W/m}^2\). A whisper ≈ 15 dB, normal conversation ≈ 60 dB, a jet engine ≈ 120 dB. A musical note has five parameters: amplitude (loudness), frequency (pitch), voice (waveform shape), envelope (time-varying amplitude shaping), and duration. Middle A = 440 Hz; other notes follow \(f = 440 \times 2^{N/12}\) Hz where N is the number of half-steps from A. Middle C = 261.6 Hz. Real instruments produce complex waveforms containing multiple harmonics — a pure tone is a perfect sine; a flute is nearly pure; a violin and human voice contain rich harmonic content. Music generation therefore requires DAC waveform tables richer than a simple sine wave.

Sine Wave Generation and SNR. A timer ISR fires at rate \(f_s\) (the sample rate), increments a table index I modulo the table length N, and calls DAC_Out(Wave[I]). For a 440 Hz tone with 64-sample table at 80 MHz clock: \(\Delta t = \dfrac{1}{440 \times 64} = 35.5\,\mu s\) per sample → set timer period to 2841 counts. SNR (Signal-to-Noise Ratio) \(= 20\log_{10}(V_{signal}/V_{noise})\), measured as an amplitude ratio in dB. For an N-bit buffer table: SNR ≈ 6N dB — 5 bits ≈ 30 dB, 8 bits ≈ 48 dB, 10 bits ≈ 60 dB, 12 bits ≈ 72 dB. Measured for a 12-bit DAC: 32 outputs gives SNR ≈ 31 dB (equivalent to ≈5 bits effective resolution); 256 outputs gives SNR ≈ 49 dB (≈8 bits). Two tones (melody + harmony) can be mixed by: (1) reducing each to 11-bit range (0–2047), (2) using two timer ISRs one per channel, (3) maintaining two separate table indices, (4) adding the two values and outputting the 12-bit sum. The 12-bit DAC uses only 11-bit waveforms precisely so the sum of two channels fits in the 12-bit range without overflow.

Audio Amplifier Circuit. The DAC output (~0–3 V) cannot drive a speaker directly — speakers are 8 Ω loads and require significant current. Two amplifier options are used: the TPA731 (class-AB, 700 mW into 8 Ω, differential bridge output) and the MC34119 (45 active transistors, differential gain = \(2 \times R_f/R_i\), Ri = 3.0 kΩ, Rf = 75 kΩ → gain ≈ 50). Both amplifiers are AC-coupled to the DAC through a series capacitor and resistor to remove the DC bias from the DAC midpoint. A bypass capacitor (CB) on the BYPASS pin of TPA731 stabilizes the internal VDD/2 reference. A reconstruction (anti-imaging) filter is needed between DAC and amplifier to smooth the staircase waveform — without it the output contains high-frequency replicas of the desired signal at multiples of the sample rate. The full hardware chain: TM4C123 (SSI1, PD0–PD3) → TLV5616 DAC → coupling cap → TPA731/MC34119 → Speaker. The LM4041 shunt reference with voltage divider (RS, R1, R2) sets the 1.5 V DAC reference.

Key Takeaways

  • Sample rate must exceed 2× the highest audio frequency (Nyquist) — for notes to 4 kHz, use at least 8 kHz sample rate
  • Never block inside the DAC ISR — must write one sample and return in under 1/fₛ seconds or audio glitches result
  • Use a volatile global to pass DAC values between ISR and main — never share non-atomic data structures across threads
  • The TLV5618 output formula is \(V_{out} = 2 \cdot REF \cdot n / 4096\) — set REF = 1.5 V with the LM4041 shunt reference to get 0–3 V range
  • Two-channel mixing: reduce each waveform to 11-bit range, add, output 12-bit sum — this is why the 12-bit DAC uses 11-bit waveforms
  • SNR ≈ 6 dB per bit of DAC resolution; 32-sample table gives only ~5-bit effective SNR even with a 12-bit DAC — buffer size matters as much as DAC resolution
  • SSI clock formula: \(f_{SSI} = f_{bus} / (CPSDVR \times (1+SCR))\) — set CPSDVR=8, SCR=0 for 10 MHz from 80 MHz bus; always verify against DAC datasheet maximum
  • Check TNF (Transmit FIFO Not Full) bit before writing SSI_DR_R — the ISR fires faster than the FIFO drains if sample rate × 16 bits exceeds SSI bandwidth
  • The MC34119 differential gain = 2×Rf/Ri; AC-couple the DAC output through a series capacitor to remove the DC bias before the amplifier input
  • Power = V²/R — measure RMS voltage across the speaker with a voltmeter in AC mode to calculate delivered power
  • DAC Figure of Merit = (Speed × Resolution) / Power — use this to compare DAC options in your design

Real-World Applications

Lookup table DAC synthesis is how video game consoles produced music in the 1980s and how modern synthesizers generate waveforms digitally. Audio DACs in smartphones use the same principle at 24-bit, 192 kHz. The TLV5618 thermometer-coded DAC architecture — resistor string with buffer — is used in precision industrial DACs for process control (4–20 mA loops) and medical infusion pumps. The TPA731 and MC34119 are class-AB linear amplifiers — the same topology as home stereo systems, scaled to milliwatts. The SSI/SPI interface timing analysis (setup, hold, propagation delay) is the same procedure used to validate any high-speed digital interface from DDR memory to PCIe.

Group 6 Analog Front-End: Op-Amps, Sensors & Filters
›

Why This Matters

The real world does not produce clean 0–3.3 V signals matched to ADC input ranges. Real sensors produce millivolt outputs buried in noise, at impedance levels that would load a GPIO flat, with common-mode interference from power supplies and radiated fields. The op-amp circuits and analog filters here are the hardware that makes sensors actually usable — they are the difference between a precision measurement system and an expensive noise generator.

Concepts

A transducer converts a primary form of energy into a corresponding signal with a different energy form. Primary forms include mechanical, thermal, electromagnetic, optical, and chemical energy. Transducers take the form of a sensor (detects/measures a signal from the real world) or an actuator (generates a signal or stimulus to affect the real world). A complete measurement system follows: measurand → sensor → signal variable → display/processing → measurement. The sensor output must relate to the measurand in a known, calibrated way.

Sensor modality refers to the category of physical input: light, sound, temperature, pressure, electromagnetic, chemical/gas, acceleration, motion, humidity, and flow are all distinct. Sensors fall into a hierarchy of intelligence: a smart sensor adds data processing and self-knowledge/communication; an intelligent sensor adds perception (recognizing and interpreting data); a cognitive sensor adds reasoning — it makes decisions, self-calibrates, self-diagnoses faults, and adapts autonomously.

Static Sensor Characteristics. Accuracy is closeness to the true value — related to bias; measured by absolute error (RESULT − TRUE VALUE) and relative error. Resolution is the minimal input change producing a detectable output change. Precision is repeatability — agreement between successive readings under the same conditions; related to variance and is a necessary but not sufficient condition for accuracy. A sensor can be high-precision but inaccurate (repeatable but biased). Other characteristics: drift, range, sensitivity (\(\partial X/\partial T\)), selectivity (responding only to the target measurand), linearity, hysteresis.

Sound — Electret Microphone. An electret microphone contains a permanently charged diaphragm and a back-plate capacitor, plus a JFET buffer transistor. The basic interface biases the JFET with R1 (2 kΩ) to Vcc and AC-couples the output through a capacitor. The full conditioned circuit for this course (Example 10.4) sets the DC bias to 1.65 V using a voltage divider R2 = R3 = 22 kΩ, amplifies with OPA2350, and feeds V₃ into the TM4C123 ADC. Key electret parameters: voltage and impedance, directional vs. omnidirectional, signal-to-noise ratio, sensitivity, and frequency response.

Distance Sensors. An ultrasonic rangefinder (e.g., HC-SR04) emits ultrasound pulses and measures the echo time Δt; distance = (speed of sound × Δt) / 2. The trigger (5 µs pulse on CCP0) and echo (measured via input capture on CCP1) are handled by TM4C123 timers. A Sharp IR distance sensor outputs a nonlinear analog voltage vs. distance — requiring a lookup table to convert ADC counts to cm.

Temperature — Thermistors vs. Thermocouples. A thermistor has resistance \(R = R_0 \exp(\beta/T)\), converted to temperature via \(T = 1/(H_0 + H_1 \ln R)\). Thermistors are more sensitive and have better resolution but are non-linear and require calibration. A thermocouple is sturdier, faster, more linear, and inert — but requires cold-junction reference compensation.

Operational Amplifiers. An op-amp is a high-gain differential amplifier with non-inverting input V_y, inverting input V_x, output V_out, and power supply ±Vs. The open-loop gain is enormous (≈ 10^6); even a tiny differential voltage saturates the output without negative feedback. Four fundamental rules for closed-loop negative feedback with unsaturated output: (1) output bounded by ±Vs; (2) input currents I_x and I_y are zero; (3) negative feedback drives V_x to equal V_y (virtual short); (4) positive feedback or no feedback drives V_out to ±Vs. A rail-to-rail op-amp (e.g., OPA2350) operates linearly all the way from −Vs to +Vs — mandatory with a 3.3 V single supply.

Key non-idealities: offset voltage Vos, offset current Ios, bias current I_b (equalize source resistance at both inputs to cancel), input impedance, output impedance. Op-amps used in this course: OPA350 (high speed, 38 MHz GBW, 122 dB open-loop gain, 10¹³ Ω input impedance — primary choice); OPA330 (low power, 350 kHz, 35 µA); TLC2272 (rail-to-rail, 2.18 MHz); MAX492 (rail-to-rail, 500 kHz). Select resistors in the 1 kΩ–1 MΩ range; circuit BW ≈ GBW / closed-loop gain.

CMRR. For an op-amp circuit, the output decomposes as \(V_{out} = G_{dm}(V^-_{in} - V^+_{in}) + G_{cm}(V^-_{in} + V^+_{in})/2\). The Common-Mode Rejection Ratio \(\text{CMRR} = 20\log_{10}(G_{dm}/G_{cm})\) — a large CMRR means noise present equally on both inputs is strongly rejected. To improve CMRR: equalize source resistance at both op-amp inputs.

Op-Amp Circuit Design Procedure. Any linear requirement can be written as \(V_{out} = A_1V_1 + \cdots + A_nV_n + B\). Design steps: (1) choose V_ref; (2) rewrite B as A_ref × V_ref; (3) add a ground input — the sum of all gains must equal +1 (solve for A_g); (4) choose feedback resistor R_f; (5) set R_i = R_f / |A_i| for each input, connecting to + for positive gain and − for negative; (6) build and verify. All op-amp pin voltages must stay within {−Vs, +Vs}. Example: V_out = 6V_in − 9, V_ref = 1.5 V → R_f = 12 kΩ, R_in = 2 kΩ, R_ref = 2 kΩ, R_g = 12 kΩ; BW ≈ 38 MHz / 6 = 6.3 MHz.

An instrumentation amplifier (INA) is a precision differential amplifier with gain set by a single external resistor: INA122P gives \(G = 5 + 200\,\text{k}\Omega / R_G\); AD627 uses the same formula. The output equation is \(V_{out} = G \cdot (V_2 - V_1) + V_{ref}\), so when the two inputs are equal the output equals V_ref — allowing a precise offset to be injected. The INA122 is a low-power, single-supply rail-to-rail device. A typical thermistor or strain-gauge interface uses a resistance bridge: four resistors in a Wheatstone configuration produce a small differential voltage proportional to the measurand, fed into the INA high-impedance differential inputs. A thermocouple interface (e.g., AD627AN) uses gain = 1000 via a 200 Ω gain resistor, followed by a 2-pole Butterworth 104 Hz LPF. An EKG front-end uses: AD627AN preamp (gain 48, 0.05 Hz HPF for AC coupling), a second OPA2350 stage (gain 48), and a 153 Hz 2-pole Butterworth LPF for anti-aliasing — total gain 2500 (preferably split as 50×50 for maximum bandwidth, rather than 10×250, to avoid saturation). Analog filters shape frequency content before digitization. A 2-pole Butterworth LPF gives −40 dB/decade rolloff with a maximally flat passband — used in Lab 8 for thermistor anti-aliasing.

Threshold Detection (Lec 38). A comparator / threshold detector converts a continuous analog signal into a binary digital output based on whether the signal is above or below a reference voltage V_t. Without feedback (no hysteresis): a simple open-loop comparator (op-amp in saturation) switches output between 0 V and V_DD as V_in crosses V_t. The problem is that near V_t, noise on the input causes rapid spurious output transitions — every noise ripple crossing the threshold produces a pulse. With positive feedback (hysteresis): feeding a fraction of the output back to the non-inverting input creates a Schmitt trigger. The threshold shifts depending on the current output state, creating two thresholds separated by the hysteresis band ΔV. For an inverting Schmitt trigger with R₁ and R₂: \(\Delta V = V_{DD} \cdot R_1 / (R_1 + R_2)\). When V_out = 3.3 V, the upper threshold V_p is higher; when V_out = 0 V, the lower threshold is reduced — the signal must travel further to trigger another transition, rejecting noise. Benefits of hysteresis (Schmitt-trigger CMOS vs. standard CMOS inputs): lower power supply noise, better noise rejection on slow-rising inputs, and stable switching on inputs with long transition times. Three key applications: (1) converting a noisy analog sensor (tachometer, optical detector) output to a clean digital square wave; (2) protecting the microcontroller GPIO from out-of-range voltages via clipping circuits — a diode to +3.3 V clamps positive excursions, a diode to GND clamps negative excursions; (3) a resistor voltage divider biases the TACH signal to the operating range of the comparator. A practical design example: TACH signal V_pp = 0.6 V riding on a 1.0 V DC level, desired hysteresis ΔV = 0.18 V, V_ref = 2.2 V. Select R21 = 3 kΩ, R10 = 30 kΩ (given). Compute R20: \(V_{ref} = 3.3 \cdot R_{21}/(R_{21}+R_{20})\) → R20 = 1.5 kΩ. Compute R2: \(\Delta V \approx 3.3 \cdot R_1/(R_1+R_2)\), with R1 = 10 kΩ → R2 ≈ 173 kΩ. Compute R11 to center the bias midpoint between V_p = 2.265 V and V_p(low) = 2.069 V (midpoint 2.167 V): \(R11 = 3.3 \cdot 30\text{k}/2.167 - 30\text{k} = 15.7\,\text{k}\Omega\). The actual computed hysteresis is 0.196 V (16 mV error from the ideal 0.18 V due to loading). Input capture records timer edge timestamps for distance and speed measurement.

Analog Filter Design (Lec 36). Analog filters serve three roles in embedded data acquisition: (1) anti-aliasing — reject all signal energy above \(f_s/2\) before the ADC (must be analog, cannot be digital); (2) noise reduction / SNR improvement — amplify desired signal and band-limit noise; (3) DC blocking (HPF) — remove unwanted DC offsets for AC signals such as sound and EKG. A typical data acquisition chain is: Physical Signal → Transducer (H₁) → Amplifier (H₂) → Analog Filter (H₃) → ADC, with total system response \(Z_2(s) = H_3(s)H_2(s)H_1(s)X(s)\). The four filter types by frequency response: Low-pass (LPF) — passes DC and low frequencies, attenuates above the cutoff; High-pass (HPF) — attenuates DC, passes above cutoff; Band-pass — HPF and LPF in series (buffer between them if passive); Band-stop / Notch — HPF and LPF in parallel followed by a summer (HPF corner must be higher than LPF corner). Filter order determines rolloff steepness: 1st-order = −20 dB/decade, 2nd-order = −40 dB/decade, nth-order = −20n dB/decade. Higher-order filters approach the ideal "brick wall" response. The 2-pole Butterworth LPF design procedure: (1) select \(f_c\); (2) compute capacitor values \(C_{1A} = 141.4\,\mu\text{F}/(2\pi f_c)\), \(C_{2A} = 70.7\,\mu\text{F}/(2\pi f_c)\); (3) find standard capacitor values C₁B, C₂B maintaining the 2:1 ratio (scale factor x = C₁A/C₁B); (4) set \(R = 10\,\text{k}\Omega \cdot x\); (5) set V_ref to analog ground (1.5 V or 0 V). The magnitude response is \(|V_{out}/V_{in}| = 1/\sqrt{1 + (f/f_c)^4}\). When R₁=R₂ and C₁=C₂, the cutoff is \(f_c = 1/(2\pi R_1 C_1)\) for both the HPF and LPF 2-pole Butterworth topologies. HPF topologies include passive RC (C in series, R to V_ref), resistor-divider bias (two 2R to supply rails), and two-input differential (useful for differential input signals). A practical audio electret filter example: HPF at 3 Hz (50 kΩ, 1 µF), HPF at 188 Hz (180 Ω, 4.7 µF), LPF at 4.8 kHz (3.3 kΩ, 10 nF) using an OPA2350 amplifier with X7R ceramics for decoupling and C0G ceramics for the signal path.

Passive Components: Resistors & Capacitors (Lec 35). Resistors are specified by value (E12/E24/E96 series), power rating, size/mounting, and tolerance. For through-hole prototyping, use 1/6 W (same footprint as 1/8 W). Keep dissipation below V²/R = I×V < 1/6 W. The E24 series provides 24 values per decade (e.g., 1.0, 1.1, 1.2, 1.3, 1.5, 1.6, 1.8, 2.0, 2.2, 2.4, 2.7, 3.0, 3.3, 3.6, 3.9, 4.3, 4.7, 5.1, 5.6, 6.2, 6.8, 7.5, 8.2, 9.1 Ω and decade multiples). For precision requirements, weigh a $2 precision resistor against a 50¢ trimmer potentiometer. Capacitors are specified by value, polarization, size, and tolerance. Four types matter in embedded design: C0G/NP0 ceramic (0.1 pF–0.47 µF, ±1%, excellent temperature coefficient) — use for analog signal processing and precision filters; X7R/Y5V ceramic (10 pF–47 µF, −20/+80%, good T coef) — use for bypass/decoupling; tantalum electrolytic (0.1–220 µF, ±10%, poor T coef) — bulk board decoupling; aluminum electrolytic (0.47 µF–0.01 F, ±20%, poor T coef) — bulk decoupling only. All capacitors are polarized except ceramics — never reverse-bias an electrolytic. Real capacitors have Equivalent Series Resistance (ESR) and Equivalent Series Inductance (ESL): ceramics have very low ESR (0.004 Ω) and ESL (200 pH) making them effective at high frequency; electrolytics have high ESR (1 Ω) limiting their filtering effectiveness above ~1 MHz. The capacitance of a dielectric capacitor follows \(C = k\varepsilon_0 A/d\) — the dielectric constant k varies significantly with temperature, especially for high-k ceramics (Y5V peaks sharply near room temperature, X7R is flatter, C0G is essentially flat). For power supply decoupling: place a 0.1 µF ceramic at each IC power pin plus a bulk 10 µF (tantalum or electrolytic) at the power entry; for digital/analog separation use a ferrite bead between digital and analog V_DD with separate decoupling on each side.

Key Takeaways

  • Accuracy (closeness to true value), precision (repeatability), and resolution (smallest detectable step) are three independent quantities — a sensor can be precise but inaccurate
  • Match the sensor output range to the ADC input range — a 0–100 mV sensor into a 3.3 V ADC wastes 97% of available resolution
  • Op-amp rules: V_x = V_y and I_x = I_y = 0 — both hold only when negative feedback is present and the output is unsaturated
  • Rail-to-rail op-amps (OPA2350) are mandatory with single 3.3 V supply — standard op-amps cannot swing near the rails
  • Equalize source resistance at both op-amp inputs to cancel bias current offset errors
  • Circuit BW ≈ GBW / gain — higher gain reduces bandwidth; select op-amp based on the GBW product needed
  • Sum of all op-amp circuit gains = +1: the ground path gain absorbs the remainder, leaving no free variables
  • Thermistors are more sensitive than thermocouples but non-linear — always use calibration tables or the Steinhart–Hart equation
  • Electret microphone must be AC-coupled to remove DC bias before feeding an amplifier or ADC
  • High CMRR is essential for sensors with long leads — ECG electrodes, strain gauges, and any sensor near switching power supplies

Power System Design (Lec 29). A power budget calculates the average current across all operating modes to determine battery life. Average current must be less than E/t_life (battery energy divided by desired lifetime). Two regulator families: linear regulators and switching (buck-boost) regulators.

Linear regulators are low-noise and easy to solder but dissipate wasted power \(P = (V_{in} - V_{out}) \cdot I_L + V_{in} \cdot I_G\). They require a dropout voltage: \(V_{in} \geq V_{do} + V_{out}\). Linear regulators available in the lab cabinet: LM78M05 (+5 V, 500 mA, V_do = 2.5 V), LM2937ET-3.3 (+3.3 V, 500 mA, V_do = 0.8 V — the standard Lab 7 choice), LP2950ACZ-3.3G (+3.3 V, 100 mA, V_do = 0.38 V). Adjustable regulators (LP2981, LM1117) use external resistors to set output voltage. Since I_out ≈ I_in for linear regulators, all wasted power heats the regulator package.

Switching regulators (buck, boost, buck-boost) achieve high efficiency by switching inductors and capacitors — P_out ≈ x × P_in where x is efficiency (often 90%+). A buck-boost (TPS63001: 1.8–5.5 V in → 3.3 V out, up to 1200 mA) is the only topology that can boost voltage — so V_out > V_in is possible. Switching regulators are noisy (large RMS ripple on V_out) and require a challenging PCB layout (switching inductor must be close to the IC to minimize loop area). The TPS63001 switches at 1.3 MHz; it needs a 2.2 µH inductor and 10 µF input/output capacitors. Low-power design techniques: sleep mode with 32.768 kHz crystal (reduces active clock to a few µA); turning off analog power when not in use (MAX604 load switch controlled by a GPIO pin separates digital and analog power rails to prevent wasted quiescent current).

Batteries. Key parameters: storage capacity (mAh), size, weight, cost. AA battery comparison: Alkaline (1.5 V, 2000 mAh, primary), Lithium (1.5 V, 3000 mAh, primary — lightest), NiCad (1.2 V, 1200 mAh, secondary), NiMH (1.2 V, 1800 mAh, secondary), Li-ion (3.6 V, 1900 mAh, secondary). Li-ion has the best energy density (Wh/kg and Wh/L) for portable products. Key design questions: if rechargeable, how will it be charged? If disposable, how easy is it to replace? Battery charger ICs: BQ24005 for Li-ion (CC/CV charging, thermistor safety monitoring, AC adapter input); MAX713 for NiMH (fast charge detection via −ΔV or temperature, PGM pins set charge rate and number of cells).

Specifications and Constraints (Lec 30). A requirements document states what the system must do — not how. It is unambiguous, complete, verifiable, and modifiable. Standard sections: Overview (objectives, process, roles, interactions, terminology, security); Function Description (functionality, scope, prototypes, performance, usability, safety); Deliverables (reports, audits, outcomes). Document conventions: must = mandatory requirement from an outside agency (legal/regulatory — failure precludes commercialization); shall = mandatory requirement specified by the company (failure may preclude commercialization); should = additional requirements addressed if time and cost allow. System specifications (what must be done) include: bandwidth, latency, accuracy, resolution, repeatability (σ under same conditions), and reproducibility (σ under different conditions). System constraints (what must NOT be done) include: power (must not exceed X mW), size (must not exceed X × Y × Z), weight, cost, memory, and time-to-market. Dynamic efficiency: bandwidth = calculations or data per second; latency = time from request to response (real-time means bounded latency). Static efficiency: size of RAM (TM4C123: 32 KiB) and ROM (TM4C123: 256 KiB). Good software style: names should have meaning, avoid ambiguity, hint at type, use prefixes to distinguish public objects, use capitalization conventions (SCREAMING_SNAKE for constants, camelCase for locals, PascalCase for private globals, Module_Function for public functions). Code comments should explain how and why, not just what. Use Doxygen-style headers (@brief, @param, @return, @note) in .h files so documentation can be auto-generated.

Key Takeaways

  • PCB layout order: place fixed parts (connectors on edges) → bypass caps near power pins → route ground first → then power → then critical signals → add test points

  • Linear regulator wasted power = (V_in − V_out) × I_out — for a 9 V battery through a 3.3 V LDO at 100 mA, you waste 570 mW as heat
  • Dropout voltage constraint: V_in must exceed V_out + V_do — a fully discharged Li-ion at 3.0 V barely meets the LM2937's V_in minimum of 4.1 V
  • Switching regulators are efficient but noisy — add a post-filter if feeding sensitive analog circuitry
  • Battery life = capacity (mAh) / average current (mA) — calculate weighted average across all operating modes including sleep
  • PCB files are due by a hard deadline — late submission means paying for your own fabrication, period
  • A requirements document states WHAT the system does, never HOW — mixing them creates unmaintainable specs
  • Must/shall/should have precise legal meanings — "shall" in a spec is a contractual obligation to your company; "should" is aspirational
  • Repeatability (same operator, same day) and reproducibility (different operator, different day) are distinct specifications used in metrology and medical device standards
  • Eliminate connectors wherever possible — every connector is a point of failure, increased cost, and signal integrity risk
  • Build a cardboard mockup before laser-cutting your enclosure — verifies PCB placement, LCD/switch cutouts, and connector clearances at zero cost
  • Gather all BOM components before Lab 7B — having parts in hand before the PCB arrives dramatically simplifies integration

Why This Matters

Motors convert electrical energy into mechanical motion — they are in every robot joint, EV drivetrain, HVAC fan, drone, hard drive, and industrial machine. Controlling them precisely requires understanding both the physics (back-EMF, inductance, torque) and the electronics (driver circuits, PWM, current sensing). Getting this wrong destroys hardware — a motor without a flyback diode will take out its driver transistor on the very first switch-off.

Concepts

DC motors convert electrical power to rotation. Torque is proportional to current; speed is proportional to voltage minus back-EMF. Back-EMF is the voltage generated by the spinning motor acting as a generator — it opposes applied voltage and grows with speed. At stall (speed = 0), back-EMF = 0 and current is maximum (Vsupply / Rwinding). This is why motors draw the most current at startup.

PWM (Pulse-Width Modulation) controls average motor power by varying the fraction of time (duty cycle) the drive signal is high: Power = Voltage × Current × DutyCycle. The TM4C123 timer generates PWM at programmable frequency and duty cycle via PWM0A_Init(period, duty): SYSCTL_RCC_R selects the 1/2 clock divider; PWM0_0_GENA_R = 0xC8 (low on LOAD, high on CMPA down); TAILR_R sets period; CMPA_R sets duty. Example: PWM0A_Init(40000, 30000) gives 1 kHz at 75% duty cycle. Change duty on the fly with PWM0A_Duty(duty) which writes only CMPA_R. PCTL value 4 maps PB6 to M0PWM0.

Driver circuits. The TIP120 Darlington (\(h_{fe} \geq 1000\)) acts as the power switch — a 3 mA GPIO drives 3 A of motor current. Base resistor R_b ≤ (VOH − V_be)/I_b = (3 − 2.5)/1 mA = 500 Ω; use 100 Ω. V_CE at saturation is ~2 V. The IRLD024 / IRLD120 Power MOSFET is the preferred driver: V_GS = 3.3 V turns it fully on; I_DS = 2.5 A max (need only 0.1 A for the Pololu motor). Gate resistor R_b = 10 kΩ (just to limit transients). From the IRLD024 datasheet at V_GS = 3.3 V: I_DS ≈ 0.1 A at V_DS ≈ 0.05 V — the MOSFET is in the linear (ohmic) region, acting like a low-value resistor. A flyback diode (1N914) across the motor clamps the back-EMF spike on every PWM turn-off. Motor characterization procedure: (1) vary supply voltage, measure current with ammeter → Power = V × I; (2) measure TACH_OUT frequency with oscilloscope → Speed (rps) = Frequency/6 for the Pololu motor; (3) plot Speed vs. Power to get the linear motor model \(m\) and time constant \(\tau\).

Input Capture — Period Measurement (Lec 39). Speed measurement uses input capture: a dedicated timer mode that automatically latches the current timer count into a capture register (TAR) on every selected edge of an external CCP pin — no polling required. The TM4C123 supports rising, falling, or both edges via the TAEVENT bits; the CAERIS flag indicates the event occurred; CAEIM arms the interrupt. Each timer (0–5 for TM4C123, plus Wide Timers) has two CCP channels: T0CCP0/T0CCP1 up to T5CCP0/T5CCP1, mapped via the PCTL register. The 16-bit timer is extended to 24 bits using the prescaler register (TAPR = 0xFF), giving resolution = 12.5 ns at 80 MHz. Since the counter runs down, period is computed as: \(\text{Period} = (\text{First} - \text{TAR}) \mathbin{\&} \texttt{0x00FFFFFF}\) — the subtraction naturally handles the single timer wrap (rollover), so no special carry logic is needed; the 24-bit mask prevents sign-extension errors. Period in units of 12.5 ns. The interrupt handler: acknowledge (clear) the flag immediately, compute Period, save current TAR as First for the next measurement, and set a Done semaphore for the main thread to consume. Time differences between same-sense edges (rising→rising) give period; between opposite-sense edges (rising→falling) give pulse width. Applications: tachometer speed (period = 1/frequency), ultrasonic echo time, capacitance measurement (RC charge time to 63.2% of Vdd).

Tachometer Design (Lec 42). The Pololu #3675 brushed DC motor includes a Hall-effect encoder. The Hall IC produces an open-drain output that is pulled to 3.3 V via a 4.7 kΩ resistor — connect the A channel to an input capture CCP pin (e.g., T1CCP0). The motor generates 12 pulses per revolution on the A channel (or 6 if only one channel used). Speed formulas: Speed (rps) = 6,666,666 / Period (in 12.5 ns units); Speed (rpm) = 400,000,000 / Period. A practical software structure: the input capture ISR computes Period and resets a "Count" variable; a periodic 10 ms timer ISR increments Count — if Count ≥ 3, the motor is considered stopped (Speed = 0). Valid periods must lie in a range consistent with the expected speed range; outliers (from noise or startup) are rejected. The motor's analog tachometer signal (sinusoidal, amplitude ≈ rps, frequency = 4× or 12× rps) requires the threshold detection circuit above to produce a clean digital signal before feeding the CCP input.

Brushless DC (BLDC) Motors (Lec 48). A BLDC motor has no brushes — commutation is performed electronically using three Hall-effect sensors (HS1, HS2, HS3) that report rotor position as a 3-bit code. The controller reads the Hall sensor code and energizes the correct pair of stator windings (phases A, B, C) by driving power transistors. Each of the six commutation steps applies +V to one phase and GND to another, leaving the third floating (HiZ). The sequence cycles through states 1→2→3→4→5→6 to rotate the motor CW; reverse the sequence for CCW. Because there are no brushes, BLDC motors achieve higher efficiency (mechanical power/electrical power), longer lifespan, high power density, precise speed control (via the controller), and quiet operation. Disadvantages include higher cost (magnets + controller + Hall sensors), requirement for an electronic control system, and limited torque at high speeds due to back-EMF limits and heat dissipation challenges. Common BLDC applications: drone ESCs, EV powertrains, computer fans, robot joints, hard disk spindle drives.

Servo Motors (Lec 48). A hobby servo motor has a built-in position controller — the embedded electronics measure shaft angle via a potentiometer and drive an internal brushed DC motor to reach the commanded angle. The interface is a single PWM signal with a 20 ms (50 Hz) period: pulse width encodes position. Minimum position (0°) = 0.5 ms pulse; center (30°) = 1.5 ms; maximum (60°) = 2.5 ms. The servo needs its own +5 V regulator (7805-style) separate from the microcontroller supply — servo load transients would crash the microcontroller's LDO. Interface circuit: V_in → 7805 → +5 V servo power; uC PWM output drives the servo Command signal directly (3.3 V logic is accepted). Software: generate a 50 Hz PWM with duty cycle matching the desired pulse width. Example: for 20 ms period at 80 MHz clock with /64 prescaler, period counter = 25000; duty = 625 (0.5 ms), 1875 (1.5 ms), or 3125 (2.5 ms). Servo advantages: simple single-wire interface, no external controller required, position precision determined by the encoder, high torque-to-inertia ratio, quiet at high speed, and vibration-free. Disadvantages: requires "tuning" of the internal feedback loop, peak torque limited to 1% duty cycle, complex encoder requirement, and motor can be damaged by sustained overload.

Stepper Motors (Lec 49). A stepper motor moves in discrete angular steps by sequentially energizing coils, achieving open-loop position control without an encoder. Typical step angle = 1.8° (200 steps/rev for a motor with 100 alternating permanent magnets). A unipolar stepper has two windings per phase (6-wire) and can reverse the magnetic pole without reversing current direction — each coil is simply switched on/off. A bipolar stepper has a single winding per phase (4-wire) and requires current direction reversal (via an H-bridge) to flip polarity — more efficient use of the copper. Full-step sequence for a 4-bit output: 0101 → 0110 → 1010 → 1001 → (repeat). Each step rotates the rotor 18°. Half-step sequence doubles the resolution to 400 steps/rev: 0101, 0100, 0110, 0010, 1010, 1000, 1001, 0001. An FSM (Finite State Machine) is the natural implementation: each state stores an output pattern and two Next pointers (CW and CCW). CW() advances the FSM pointer and writes the new state output to the port; CCW() traverses the reverse pointers. A shaft position variable (POS, 0–199 for full-step, 0–399 for half-step) tracks cumulative steps. Driver ICs: 7406 (40 mA, unipolar), 75492 (250 mA), 2N2222 (600 mA), L293 (1 A/channel, both unipolar and bipolar), DRV8848 (H-bridge with enable/direction interface). The DRV8848 truth table: AIN1=0,AIN2=0 → Coast; AIN1=0,AIN2=1 → Reverse; AIN1=1,AIN2=0 → Forward; AIN1=1,AIN2=1 → Brake. Stepper advantages: stable under varied loads, no feedback required (motor is its own position transducer), inexpensive, excellent low-speed torque, excellent repeatability, and overload-safe (motor stops rather than breaking). Disadvantages: low efficiency (draws power regardless of load), torque drops rapidly with speed, prone to resonances (use micro-stepping), no missed-step detection, and very noisy at speed.

Key Takeaways

  • PWM frequency should exceed 20 kHz for quiet motor operation — audible whine occurs at lower frequencies
  • Always place a flyback diode across the motor — back-EMF spikes occur every PWM cycle and will destroy the driver transistor
  • Stall current = Vsupply / Rwinding — can be 10× running current; always size the driver for stall conditions
  • Stepper motors lose steps if commanded faster than load torque allows — use acceleration ramps for large inertia loads
  • Input capture gives sub-microsecond speed measurement resolution at 80 MHz timer clock
  • BLDC commutation: read 3 Hall sensors → select one of six transistor switch states → advance on each sensor edge
  • Servo command: 50 Hz PWM, 0.5 ms = minimum angle, 1.5 ms = center, 2.5 ms = maximum; use a separate +5 V regulator
  • Full-step sequence: 0101 → 0110 → 1010 → 1001; half-step doubles resolution to 400 steps/rev
  • Stepper FSM: CW advances state pointer forward, CCW reverses it; track absolute position in a counter variable

Real-World Applications

PWM motor control is used in every electric vehicle (Tesla uses three-phase PWM inverters at 10–20 kHz), every drone (ESCs with BLDC motors), and every computer fan. Stepper motors position print heads in inkjet printers and extruders in every 3D printer. Servo motors actuate every joint in industrial robot arms. The PI motor controller in Lab 9 is structurally identical to field-oriented control in EV drives — just at lower power.

Group 9 Control Systems
›

Why This Matters

A motor without feedback is open-loop — it goes where commanded but has no way to self-correct for load changes, friction, or component variation. Closed-loop control continuously measures the actual output and adjusts the drive to minimize error. Cruise control holding speed on a hill, a drone hovering in wind, a hard drive head landing on track — all are closed-loop systems. The PI controller in Lab 9 is the same mathematical structure used in industrial servo drives running hundred-kilowatt machinery.

Concepts

A feedback control system measures the actual output (process variable), compares it to the desired value (setpoint), computes an error, and adjusts the plant input to reduce the error. The plant is the physical system being controlled — a DC motor with its mechanical load. Its transfer function is \(H(s) = m/(1+s\tau)\) where m is the steady-state gain (speed per unit power) and τ ≈ L/R is the motor electrical time constant. The controller G(s) is in series, with feedback closing the loop: \(X(s)/X^*(s) = G(s)H(s)/[1+G(s)H(s)]\).

A proportional (P) controller computes: \(\text{output} = K_p \cdot e\). It responds immediately to error but exhibits steady-state error — a persistent offset that cannot be eliminated because at steady state, error → 0 means output → 0, which cannot overcome friction or load torque. Increasing Kp reduces steady-state error but eventually causes oscillation.

A PI (Proportional-Integral) controller adds: \(\text{output} = K_p \cdot e + K_i \sum e\). The integral accumulates past errors — if any error persists at steady state, the integral grows until the output is large enough to eliminate it, achieving zero steady-state error. In discrete time on the TM4C123: \(\text{output}[n] = K_p\, e[n] + K_i \sum_{i=0}^{n} e[i]\), computed every 10 ms in a SysTick ISR. Excessive Ki causes integral windup and oscillation; clamp the integral when the output saturates (anti-windup). A PID controller adds a derivative term: \(U_d = K_d(e[n]-e[n-1])/\Delta t\) — this anticipates future error by reacting to the rate of change, reducing overshoot. Derivative action also increases response time, so the full PID: \(U = K_p e + K_i \sum e \Delta t + K_d \Delta e / \Delta t\). Controller comparison: Open-Loop = smallest response time / highest overshoot / large error; Proportional = small response / large overshoot / small error; Integral = slower response / more overshoot / zero error; Derivative = increases response time / decreases overshoot / small change in error.

Practical implementation for the Pololu motor (Lec 44): Speed in 0.1 rpm = 133,333,333 / Delta (where Delta is the input capture period in 12.5 ns units). The control loop runs every 10 ms (100 Hz via SysTick or Timer2): compute Rot = 133333333/Delta; E = Xstar − Rot; I = I + (Ki1 × E)/Ki2 (clamped to 300–39900); P = (Kp1 × E)/Kp2 (clamped to ±1000); U = P + I (clamped to 300–39900); PWM0A_Duty(U). The Ki and Kp coefficients are expressed as integer fractions (Ki1/Ki2, Kp1/Kp2) to allow fixed-point arithmetic without floats. The loop must run at least 10× faster than the motor time constant τ. Response tuning: increase Ki1/Ki2 for more responsiveness; decrease for more stability. Four observable behaviors: optimally-damped (fast, no overshoot), over-damped (slow approach), under-damped (overshoot and ringing), unstable (growing oscillation — reduce Ki immediately).

Tuning approach: increase Kp until the motor tracks the setpoint quickly without oscillating. Then add Ki to remove the remaining steady-state offset. Increase Ki slowly — too much causes oscillation that Kp alone cannot damp. The control loop must run at a rate at least 10× faster than the plant bandwidth for stability.

Key Takeaways

  • P-only control always has steady-state error — integral action is required to achieve zero offset
  • Run the control loop at a fixed rate via timer interrupt — variable timing at high gains causes instability
  • Anti-windup is essential: stop accumulating the integral when output saturates at 0% or 100% duty cycle
  • Tune Kp first for speed, then Ki for accuracy — adding both simultaneously makes instability diagnosis impossible
  • The control loop must run at least 10× faster than the plant bandwidth — for a DC motor this means > 100 Hz

Real-World Applications

PI controllers are the most widely deployed control algorithm in the world — over 90% of industrial control loops use PID or PI. Every thermostat, cruise control, robot joint, and motor drive uses feedback control. The anti-lock braking system in your car runs a PI loop on each wheel at hundreds of Hz. Autopilots run cascaded PI loops for altitude, speed, and attitude. The voltage regulator on your laptop motherboard runs a PI loop at MHz switching frequencies. This material is prerequisite to ECE 445M (Real-Time Operating Systems).

Group 10 Communication Theory & FFT
›

Why This Matters

Every wireless system — Wi-Fi, Bluetooth, LTE, GPS, radar — is built on signal processing mathematics. The FFT is the algorithm that makes real-time spectrum analysis possible on embedded hardware. It lets you see the frequency content of a signal without an expensive spectrum analyzer — which frequencies are present, what the noise floor is, and whether harmonics indicate distortion. Sound input, frequency analysis, and digital communication are different faces of the same underlying theory.

Concepts

The Discrete Fourier Transform (DFT) converts N time-domain samples into N frequency-domain coefficients. The kth coefficient corresponds to frequency \(k \times f_s/N\) Hz and represents the signal amplitude and phase at that frequency. A 16-point DFT at 1600 Hz gives bins at 0, 100, 200, ... 1500 Hz. The DFT reveals which frequencies are present — essential for audio processing, channel characterization, and fault detection in mechanical systems.

The Fast Fourier Transform (FFT) computes the DFT in \(O(N \log N)\) operations instead of \(O(N^2)\) — for N = 1024 this is a 100× speedup. The FFT exploits symmetry in the DFT butterfly structure to eliminate redundant multiplications. A 16-point FFT runs in real time on the TM4C123. The provided FFT16.xls tool visualizes FFT magnitude for lab use. Important design factors: sampling rate f_s determines the maximum observable frequency (0 to ½f_s); number of samples N determines frequency resolution Δf = f_s/N. Example: f_s = 16 kHz, N = 16 → Δf = 1 kHz per bin. The DFT formula is \(A_k = \sum_{n=0}^{N-1} a_n W_N^{kn}\) where \(W_N = e^{-j2\pi/N}\). \(A_0\) equals the sum of all samples (DC component). For k = 1 to ½N the magnitude = |A_k|/(N/2) and phase = arctan(Im(A_k)/Re(A_k)); in dBFS: 20·log₁₀(Magnitude/FullScale). Implementation on TM4C123: fill x[] with N ADC samples (imaginary part = 0), call cr4_fft_1024_stm32(y, x, 1024), extract real/imag from y[] and compute |A_k| = sqrt(real²+imag²).

Sound input digitizes a microphone signal with the TM4C123 ADC. The electret microphone uses a diaphragm (primary sensing element) and a JFET inside the capsule (secondary conversion element) powered via a 10 kΩ resistor to +3.3 V. The interface circuit: HPF (14 Hz cutoff, C1 and R2‖R3) rejects DC; adds a 1.65 V DC offset (R2/R3 divider) to center the signal in ADC range; op-amp gain = 1 + R6/R5 = 201; LPF (3600 Hz cutoff, C3/R6) prevents aliasing. The ADC ISR (triggered by Timer_A at f_s) reads the 12-bit result, subtracts the DC offset of 2048 counts, and applies a noise rejection filter. Microphone noise is random (i.i.d.) and does not correlate with itself; the sound signal is highly correlated with itself. Autocorrelation \(R_{xx}(m) = \frac{1}{N}\sum_{n=0}^{N-1} x(n) \cdot x(n-m)\) exploits this: the running approximation \(R_{xx2} \approx \frac{K-1}{K} R_{xx2} + \frac{1}{K} x(n) \cdot x(n-2)\) tracks whether the current signal is self-correlated (periodic sound) or not (noise). Multiplying the ADC result by a scaled version of R_xx2 amplifies periodic signals and suppresses random noise — the NoiseReject() function implements this. Results show near-zero output for pure noise and faithful reproduction for tone signals. Cross-correlation \(R_{xy}(m) = \frac{1}{N}\sum x(n) \cdot y(n-m)\) compares the live input against a stored template y: large positive R_xy = same shape, 0 = unrelated, large negative = inverted. For pitch recognition (FSK): record templates of each tone; compute R_xy for each; the largest R_xy above a threshold identifies the detected pitch.

Communication theory (Lec 45): A network is a collection of interfaces sharing a physical medium and data protocol. Communication transmits energy that encodes information: Transmitter → Encoder → Channel → Decoder → Receiver. Channel impairments: attenuation (signal amplitude decreases with distance), distortion (frequency-dependent attenuation), and noise (additive random signal). Key terms: simplex = one direction only (TV broadcast); half-duplex = either direction but not simultaneously (police radio); full duplex = both directions simultaneously (telephone). Bandwidth has multiple definitions — most practically: the range of frequencies passed by the channel (Leon-Garcia). To maximize data rate: send M = 2^m amplitude levels per pulse at the Nyquist rate (2W pulses/second for bandwidth W), giving an apparent rate r = 2·W·m bits/s. But noise limits useful m — the Shannon channel capacity \(C = W \log_2(1 + \text{SNR})\) bits/s is the theoretical maximum. Example: telephone POTS (W = 3.4 kHz, SNR = 38 dB ≈ 6310) gives C = 42.9 kb/s. SNR in dB = 10·log₁₀(average signal power / average noise power). Digital repeaters overcome distance limits by regenerating a clean signal at each hop — unlike analog repeaters that amplify noise too. Frequency Shift Keying (FSK) uses sound to carry bits: 0 = 2 kHz tone, 1 = 1 kHz tone; a 16 kHz ISR fetches DAC samples and advances an index by Bit (0 = silence, 1 = 1 kHz step, 2 = 2 kHz step); a 100 Hz ISR reads one bit from a FIFO and sets Bit accordingly. Decoding uses the DFT: sample the ADC at 16 kHz; compute A₁ (1 kHz bin) and A₂ (2 kHz bin) magnitudes each millisecond; if |A₁| > threshold and |A₂| low → bit = 1; if |A₂| > threshold and |A₁| low → bit = 0. Amplitude Shift Keying (ASK) encodes bits by switching a carrier on/off. I2C encodes 0 as 0 V (output driven low, E=0) and 1 as HiZ (output floating, I=0); a pull-up resistor provides the high state. Lower pull-up resistance increases current on the high state, enabling higher speed. Data storage stores a bit as energy transmitted across time — the same principle as communication across space.

Key Takeaways

  • DFT bin k = frequency \(k \times f_s/N\) Hz — for 16-point DFT at 1600 Hz: bin 1 = 100 Hz, bin 8 = 800 Hz
  • Apply a window function (Hanning, Hamming) before the FFT to reduce spectral leakage from signal discontinuities at block edges
  • FFT magnitude = √(real² + imag²) — phase is often discarded for spectrum display applications
  • Averaging multiple FFT frames reduces the noise floor by 3 dB for each doubling of averages
  • Frequencies above \(f_s/2\) are mirrors of the lower bins — always anti-alias filter before ADC input
  • Shannon capacity C = W·log₂(1+SNR) bits/s — telephone (W=3.4 kHz, SNR=38 dB) gives only ~43 kb/s maximum
  • Autocorrelation distinguishes periodic sound from random noise — noise doesn't correlate with itself; signals do
  • FSK decode: compute DFT magnitude at each candidate frequency per bit period; largest magnitude above threshold identifies the bit
  • I2C encodes 1 as HiZ (float) not +V — saves energy; lower pull-up resistance trades power for higher speed
  • Digital repeaters restore a clean signal at each hop; analog repeaters amplify accumulated noise — digital wins on long links

Real-World Applications

The FFT is one of the most important algorithms in engineering — it enables MP3 audio compression (MDCT), OFDM modulation in Wi-Fi and 5G, radar signal processing, seismic analysis, and MRI image reconstruction. Guitar tuner apps run an FFT on the microphone input to identify the dominant frequency. The same 16-bin display you build for the LCD runs on supercomputers analyzing gravitational wave signals from the LIGO observatory — same mathematics, vastly different scale.

Group 11 Real-Time Systems
›

Why This Matters

An embedded system that gives the right answer too late is wrong. Real-time does not mean fast — it means predictable. A pacemaker that fires 10 ms late is a medical emergency. An anti-lock brake system that responds 100 ms late causes a crash. A motor control loop that misses a deadline allows instability to grow. This distinction between real-time and merely fast is what separates firmware that works reliably in the field from firmware that only works in the lab under ideal conditions.

Concepts

A real-time system is one where correctness depends both on logical results and the time at which they are produced. More precisely: a system whose behavior is constrained by operational deadlines. Two primary categories: hard real-time — missing any deadline is catastrophic and unacceptable (flight control, aviation collision avoidance, pacemakers, safety-critical plant control — the Airbus A-320 required a maximum failure probability of 10⁻¹⁰ per flight hour; GPS systems target 10⁻⁶ to 10⁻⁷); soft real-time — missed deadlines degrade quality or carry an economic penalty, but do not pose critical risk (streaming media decoders, set-top boxes, microwave ovens). The distinction between hard and soft is driven almost entirely by the potential risks and costs of a missed deadline. In practice, the optimal operating point for a soft real-time system is determined by the market. Hard real-time systems are typically found in highly regulated areas (FAA, FDA) and are often also fault-tolerant systems.

Task types. Tasks in a real-time system are classified by their arrival pattern. Periodic tasks enter execution at regular intervals — they may be hard, soft, or non-real-time. Each periodic task is modeled as a triplet (E, T, D): execution time E, period T, and deadline D (usually D = T). Aperiodic tasks arrive unpredictably and are typically soft real-time. Sporadic tasks also arrive unpredictably but are hard real-time — for example, a flight safety system responding to sudden cabin pressure loss. Tasks are further classified as critical (system fails if any deadline missed), dependent (occasional misses acceptable), or flexible (performance degrades proportionally to misses). A thread is a lightweight unit of execution contained within a process; threads in the same process share resources. Unlike processes in a multitasking OS, threads have low context-switch overhead — critical for RTOS responsiveness.

Scheduling approaches. The RTOS scheduler determines which task runs at each moment — its goal is feasibility, not fairness. If one task is due in an hour and another in a day, you work on the urgent one first. Three scheduling algorithm families: Clock-Driven (offline) — all task parameters known in advance; schedule computed offline; minimal runtime overhead; inflexible. Weighted Round Robin — tasks served in FIFO order with time quanta proportional to weight; good for network switches and QoS guarantees; not suitable for precedence-constrained tasks. Priority-Based (preemptive) — each task has a priority; the scheduler always runs the highest-priority ready task; lower-priority tasks are preempted. Priority may be static (assigned at design time) or dynamic (reassigned at runtime).

Priority scheduling algorithms. Three standard policies for periodic tasks: Earliest Deadline First (EDF) — dynamic priority; task with the nearest absolute deadline runs first; optimal in the sense that if any feasible schedule exists, EDF finds one. Least Slack Time First (LSF) — dynamic priority; slack = (time remaining until deadline) − (execution time remaining); least slack = most urgent; responsive to variable execution times. Rate-Monotonic Scheduling (RMS) — static priority; priority inversely proportional to period (shorter period = higher priority); provably optimal among fixed-priority algorithms. The RMS schedulability bound: a set of N periodic tasks with deadlines equal to periods is schedulable if \(U = \sum_{i=1}^{N} E_i/T_i \leq N(2^{1/N} - 1)\), which approaches \(\ln 2 \approx 0.693\) as N → ∞. Practically: keep CPU utilization below ~70% and all tasks with hard deadlines will meet them. To use RMS in practice, the scheduler must gather per-task data: number of tasks, resource requirements, release time, execution time, and deadlines; plus platform data: maximum context-switch time, maximum interrupt service latency, and maximum communications delays.

Cyclic Executive (CE). The simplest scheduling approach — no OS overhead. A single infinite loop calls each task function in sequence: while(1) { task1(); task2(); task3(); }. Advantages: simple implementation, very low overhead, perfectly predictable, no context-switch cost. Disadvantages: cannot handle sporadic events (everything executes in lockstep), all timing must be manually managed, and adding tasks requires redesigning the entire loop. The CE is appropriate for simple, well-characterized systems with no aperiodic requirements. When sporadic events or resource sharing are needed, an RTOS is required.

Priority inversion. A classic RTOS pathology: a low-priority task holds a resource lock (e.g., a mutex on shared memory) needed by a high-priority task. The high-priority task blocks waiting for the lock — it effectively runs at the low priority of the lock holder. If a medium-priority task that does not need the lock preempts the low-priority lock holder and runs for a long period, the high-priority task is starved for an arbitrarily long time. Solutions: Priority Inheritance Protocol (PIP) — the lock holder temporarily inherits the priority of the highest-priority waiter while holding the lock; Priority Ceiling Protocol (PCP) — each resource is assigned a ceiling priority equal to the highest priority of any task that may lock it; a task can only acquire the lock if its priority exceeds the ceiling of all currently locked resources. Priority inversion famously caused the Mars Pathfinder rover to reset repeatedly in 1997 — diagnosed and patched remotely via the Priority Inheritance Protocol.

A race condition occurs when system behavior depends on the relative timing of concurrent events — a non-deterministic bug that appears and disappears based on timing. Race conditions on shared data produce data hazards: RAW (read-after-write), WAW (write-after-write), WAR (write-after-read). All require atomic access to prevent corruption — through interrupt disabling, hardware atomic operations, or lock-free data structures. Jitter is the variation in actual execution timing from the ideal — must be bounded in hard real-time: −k ≤ δᵢ ≤ +k for all i. Engineering ethics: software failures in safety-critical real-time systems have killed people — the Therac-25 race condition, the Boeing 737 MAX MCAS, the Ariane 5 overflow. Engineers have an ethical obligation to understand failure modes, test corner cases, and communicate limitations honestly. Validating hard real-time performance is a complex undertaking: formal methods are powerful but too complex for most teams; the practical alternative is extensive testing combined with conservative resource utilization (over-design so that the system runs well under capacity).

Key Takeaways

  • Real-time means predictable, not fast — a system responding in exactly 10 ms every time is more real-time than one averaging 1 ms with outliers
  • Hard vs. soft real-time is determined by the cost of a missed deadline — planes falling from the sky vs. slightly overcooked microwave popcorn
  • Task triplet (E, T, D): execution time, period, deadline — for most periodic tasks D = T
  • Cyclic Executive is the simplest scheduler — zero overhead, perfectly predictable, but cannot handle sporadic events or flexible timing
  • Rate-monotonic scheduling: assign highest priority to shortest period — provably optimal for fixed-priority preemptive periodic tasks
  • RMS schedulability: keep total CPU utilization \(U = \sum E_i/T_i \leq 0.693\) and all deadlines are guaranteed to be met
  • EDF is optimal for dynamic scheduling — if any feasible schedule exists, EDF finds it; but static RMS is easier to analyze and validate
  • Priority inversion: a low-priority lock holder blocks a high-priority waiter — fix with Priority Inheritance (lock holder inherits waiter's priority)
  • Race conditions are non-deterministic — they may pass all lab testing and only appear in the field under specific timing
  • Hard real-time systems are intentionally over-designed — running well below the 69.3% utilization bound leaves margin for worst-case interrupt storms

Real-World Applications

Real-time systems theory governs every safety-critical embedded design: avionics flight controllers (DO-178C requires formal deadline analysis; Airbus A-320 required 10⁻¹⁰ failure probability per flight hour), automotive ECUs (AUTOSAR defines task scheduling; ABS runs PI loops at hundreds of Hz), and FDA-regulated medical devices (formal software hazard analysis required). FreeRTOS, VxWorks, and QNX implement rate-monotonic and EDF scheduling for complex products. The Mars Pathfinder priority inversion bug (1997) was one of the most famous real-world RTOS failures — it reset the rover repeatedly before engineers diagnosed it remotely and enabled the Priority Inheritance Protocol. The Therac-25 race condition and Boeing 737 MAX MCAS failures are canonical examples of what happens when safety-critical software is insufficiently validated. In industry, hard real-time systems are deliberately over-designed: keep utilization well under 70%, use conservative worst-case execution time estimates, and never assume average-case timing. This material is the direct foundation for ECE 445M (Real-Time Operating Systems).
